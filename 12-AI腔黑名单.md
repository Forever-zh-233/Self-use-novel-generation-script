# AI 腔黑名单

写手和评审都必须读。以下问题一旦出现，优先修改。

## 语言层

1. “仿佛/似乎/好像”连续出现。
2. “无法言喻”“复杂情绪”“命运齿轮”一类泛化表达。
3. 每段都有总结句。
4. 句子过度工整、对称、排比。
5. 形容词堆叠，缺少具体动作。
6. 为凑字数重复同一情绪、动作、景物或解释。
7. 碎句念经：几乎每句都独占一行、满屏短句，平推叙事不用中句承接，节奏从头到尾一个调。短句是砸节奏的工具，通篇用就成了廉价伪文学腔。

## 不信任读者层（重点盯，最容易犯）

读者是成年人，能自己拼线索、读潜台词、感受压迫和悲伤。下面全是“怕读者 get 不到、停下来替他确认”的表现，一律删：

1. 替读者下结论：摆完细节又补一句总结（“这不是巧合”“同一个人”“信息差在他这边”“她在确认一件事”）。摆完就走。
2. 先否定再给答案成瘾：反复用“不是A，是B”替读者排除错答案、钦定正确答案。偶尔点睛可以，成了每段都来的口头禅就是替读者思考。
3. 把手法名写进正文：标注“这是陈述句”“这是疑问句”。话有没有力靠话本身，不靠贴标签。
4. 写完又解释：写了动作就别解释动作的含义，写了对话就别翻译言外之意。留白是根本不说破，不是说破了再装没说。

判断方法：这句删掉，读者会不会漏掉一个他自己绝对想不到的硬信息？不会就删。

## 情绪层

1. 直接解释角色情绪。
2. 替角色讲大道理。
3. 明明该沉默，却长篇独白。
4. 情绪高潮后没有缓冲。

## 对话层

1. 所有人都说正确的废话。
2. 对话后每句都接“他说道/她说道 + 动作”。
3. 角色互相解释彼此已经知道的信息。
4. 反派长篇自曝。
5. 小孩说大人话，动物说人话。
6. 角色互相解释彼此已经知道的信息。
7. 角色说话像任务说明或剧情摘要。

## 结构层

1. 每章像作文，有开头总结、中间论证、结尾升华。
2. 冲突太抽象，没有具体人、具体事、具体代价。
3. 章末钩子只靠“更大的危机来了”这种空话。
4. 场景切换靠大段说明，不靠动作或时间地点切入。
5. Editor 把文字修得过于平滑、工整、对称，没有残句和毛刺。

## 网文体验层

1. 一章没有明确期待。
2. 没有小爽点或小反转。
3. 主角没有新选择、新麻烦、新收益或新关系变化。
4. 信息全讲完，不留下一章的钩子。
5. 章末钩子只有“更大的危机来了”，没有具体的人、声音、动作或物件。
6. 本章删掉五分之一后剧情几乎不变，说明有注水。

## 修改优先级

1. 先删总结句和替读者下结论的话。
2. 再看节奏：长则压短，碎则用中句承接——目标是长短起伏，不是越短越好。
3. 再把抽象情绪换成动作。
4. 最后检查每章是否有期待和钩子。
5. 检查章末最后 300 字，必须有具体钩子，不要抽象预告。
6. 检查对话，删掉互相解释已知信息的台词。
