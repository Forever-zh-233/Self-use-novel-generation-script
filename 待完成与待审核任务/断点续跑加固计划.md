# 断点续跑加固计划（章内阶段缓存 + 重试加固）

> 目标：单章因网络抖动（`Remote end closed connection without response`）失败重试时，
> **从断点阶段续上，而不是整章推倒重来**。第26章就是典型：writer/gate/reviewer 第一次都成了，
> 只 archivist 因网络断连失败，却整章重跑浪费 ~6 万 tok。

## 一、根因（已确认）

- 重试循环在 [run_pipeline.py:1252-1277](scripts/run_pipeline.py#L1252-L1277)：`for attempt` 里直接调
  **整个 `run_one_chapter`**，无阶段级缓存 → 任一阶段失败，整章从 writer 重跑。
- 进度条从 14% 重爬，让人误以为"全文重写"，其实是整章作废重来。
- 两次 `Remote end closed` 都是 mimo 网关单方掐连接，非内容/代码 bug。
- `per_chapter_retries` 默认 2，`retry_backoff` 默认 30s（[run_pipeline.py:1236-1237](scripts/run_pipeline.py#L1236-L1237)）。

## 二、关键不变量（已核实，是缓存方案的地基）

1. `call_role` **只在成功时**写 `output_path`（[api.py:274](scripts/pipeline/api.py#L274)）→ **产物文件存在 == 该阶段成功**。
2. `cleanup_chapter_artifacts` **只在 `run_one_chapter` 成功收尾时**调用（[run_pipeline.py:892](scripts/run_pipeline.py#L892)）；
   失败章节不清理（[core.py:359](scripts/pipeline/core.py#L359) 注释"失败章节不调用本函数，保留现场"）→ **重试期间产物都还在**，可安全复用。
3. 产物路径：
   - `manuscript_path(ch)` = `输出/文章/第NNN章.md`（[core.py:93](scripts/pipeline/core.py#L93)）—— **正文最终落盘点，在 archivist 之前的最后一步**（[run_pipeline.py:819](scripts/run_pipeline.py#L819)）。它存在 == writer+gate+reviewer+editor+factcheck 全done。
   - `role_artifact("writer",ch,"draft.md")` / `"final.md"`、`("reviewer",ch,"review.md")`、`("editor",ch,"edited.md")`、`("reviewer",ch,"review_verdict.json")`。
4. `recover_state_from_completed_articles`（[run_pipeline.py:178](scripts/run_pipeline.py#L178)）已能处理"正文超前于 latest_chapter"→ 启动时自动补台账。**第26章现状正属此例：直接重跑即可，无需手动清理。**

## 三、阶段顺序（run_one_chapter，确认续跑切点）

```
1 writer 构建上下文 → 2 writer 写初稿(draft.md, ~37k tok, 最贵之一)
→ 3 gate 硬检查(纯代码,免费,确定性) → 4 reviewer 评审(review.md) + verdict(review_verdict.json)
→ 5 editor 修稿(条件,edited.md→final) → final_gate(代码)
→ 6 fact_checker(条件,多轮点修) → 清洗标题 → 写 final.md + 写 manuscript(文章/)
→ 7 archivist 更新台账(~18k tok) → seed维护 → write_score_report → cleanup
```

## 四、改动方案

### 改动 A：章内阶段缓存（核心，"断哪续哪"）

在 `run_one_chapter` 开头加一个开关读取，缓存粒度由粗到细，**优先粗粒度**（收益最大、最安全）：

**A1（最大收益）——manuscript 已存在则跳过整个生成段，直奔 archivist：**
```python
resume = bool(run_cfg.get("resume_partial_chapter", True))  # 默认开
min_chars = int(run_cfg.get("min_recover_article_chars") or 1000)
manuscript = manuscript_path(chapter)
have_final_text = manuscript.exists() and len(re.findall(r'[一-鿿]', read_text(manuscript))) >= min_chars
```
- 若 `resume and have_final_text`：
  - `final = read_text(manuscript)`
  - verdict 来源优先级：读 `review_verdict.json`（存在则 `json.loads`）→ 否则 `parse_review_verdict(read_text(review.md))` → 否则 `{"total":None,"needs_revision":False,"blockers":[],"source":"resume_default"}`
  - 打印 `[续跑] 第N章正文已存在，跳过生成阶段，直接补台账。`
  - **跳过 step1~6，直接进 archivist 段（step7）**。
  - 注意：跳过段里原本写的 `final_gate.json` 等不再重算——可接受（正文已定稿）。
- 这一条就能让"第26章式失败"（只 archivist 挂）几乎零浪费。

**A2（细粒度，生成段内逐阶段复用）——manuscript 尚未生成时：**
对每个 LLM 阶段，包一个 helper：
```python
def cached_or_call(role, chapter, suffix, gen_fn):
    p = role_artifact(role, chapter, suffix)
    if resume and p.exists():
        txt = read_text(p).strip()
        if txt and len(txt) >= 50:   # 简单有效性门槛
            cli_print(f"[续跑] 复用已有 {role}/{suffix}")
            return txt
    return gen_fn()
```
- **draft**（step2）：`draft.md` 存在且有效 → 复用，跳过 writer 写初稿（最贵）。
  - 注意 best_of_n 路径也最终写 `draft.md`（[run_pipeline.py:535](scripts/run_pipeline.py#L535)）→ 复用 draft.md 时**整组 best_of_n 都跳过**，正确。
- **gate**（step3）：纯代码、确定性、免费 → **不缓存，照算**（拿 draft 重算即可）。
- **review**（step4）：`review.md` 存在 → 复用；verdict 随之 `parse_review_verdict`。
- **editor**（step5）：`edited.md` 存在 → `final = read_text(edited.md)`，跳过 editor 调用。
  - 但"是否需要修稿"的判断仍按 `gate.passed / verdict.needs_revision`；若判定不需修稿则 final=draft，不读 edited。
- **fact_check**（step6）：多轮点修、状态复杂 → **本轮不做缓存，允许重跑**（它是幂等型点对点修改，且不是主成本）。
  - 可选加分：若已有 `final.md` 则视为 factcheck 完成、直接复用 `final.md`（注意 final.md 在 factcheck 之后、manuscript 之前写，[run_pipeline.py:818](scripts/run_pipeline.py#L818)）。优先级低，先不做也行。

**实现要点：**
- 所有"读缓存"都要做有效性门槛（非空 + 最小长度），坏文件不能当成功。
- 跳过 step1 的"构建上下文"时也要小心：A1 整体跳过即可；A2 复用 draft 时 writer_input 不必再构建（省掉 `build_writer_input` 内可能触发的压缩 LLM 调用）→ 复用 draft 分支里**不要**调 `build_writer_input`。
- `write_score_report(chapter, verdict)` 依赖 verdict，务必保证三条 fallback 链可用。

### 改动 B：重试加固（不改变"停机不跳过"安全语义）

- `per_chapter_retries` 默认 2 → **3**（[run_pipeline.py:1236](scripts/run_pipeline.py#L1236)），或在 `run.json` 配 `"per_chapter_retries": 3`。
- 退避已是 `retry_backoff * attempt`（线性递增，[run_pipeline.py:1285](scripts/run_pipeline.py#L1285)）→ 保持；可把 `retry_backoff` 默认提到 45s 给网关更多缓冲。
- 配合改动 A：第 2/3 次尝试因为缓存命中，几乎只重跑断掉的那一两个阶段，重试代价骤降，多给次数也不肉疼。
- **archivist 内层 2 次重试**（[run_pipeline.py:829](scripts/run_pipeline.py#L829)）保持不动；它失败抛错后外层重试 + 改动A的 manuscript-skip 已能轻量续上。

### 改动 C：配置项（run.json）
新增/确认：
```json
"resume_partial_chapter": true,
"per_chapter_retries": 3,
"retry_backoff_seconds": 45
```
（`resume_partial_chapter` 给个开关，万一要强制全新生成可关掉。）

## 五、测试（tests/scenario_test.py 加 section）

1. **manuscript 已存在 → 跳过生成段**：seed `输出/文章/第NNN章.md`（>min_chars）+ `review_verdict.json`，
   mock/拦截 writer 调用计数=0，断言直接进 archivist；verdict 来自 verdict.json。
2. **draft.md 已存在 → 跳过 writer 写初稿**：seed draft.md，断言 writer 写初稿不被调用，但 gate 照常重算。
3. **review.md 已存在 → 跳过 reviewer**：断言复用且 verdict 由 parse 得到。
4. **edited.md 已存在 → 跳过 editor**：断言 final=edited 内容。
5. **坏缓存（空/过短文件）→ 不复用、正常重生成**：写一个 10 字的 draft.md，断言仍触发生成。
6. **resume 关闭时全部重生成**：`resume_partial_chapter:false`，断言不读任何缓存。
7. verdict 三级 fallback：verdict.json 缺失但 review.md 在 → parse；两者都缺 → 默认 verdict 不崩。
