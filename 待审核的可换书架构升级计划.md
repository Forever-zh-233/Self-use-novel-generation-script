## 四、两个"一劳永逸"的预处理生成器

这两个是"第一次写书才用、后期不碰"的一次性脚本，对应架构的预处理层。

### 生成器 1：设定生成器（新增 `scripts/generate_setting.py`）
**职责**：仿照原文的类型与质感，一次性生成本书的世界观、修炼制度、角色框架、故事核。产出后人工微调即可，几百章都用它。

- 输入：原文（仅供"学类型/学质感"，不抄设定）+ 用户的一句话立意（如"游方郎中靠了愿系统闯荡"）
- 流程：多次调 LLM —— 先定故事核 → 世界观 → 修炼制度 → 主要角色 → 卷纲骨架
- 产出：09/02/03/10 等设定文档 + 初版 book.config.json
- 铁律：**只学原文的"写法/类型/爽点结构"，绝不抄原文人名地名设定**（和 analyst 同源的防污染原则）
- 频率：每本书一次

### 生成器 2：手法分析器（analyst，提示词已改好，缺脚本入口）
**职责**：LLM 分批读完整原文，提炼**纯写作手法**（情感高潮怎么写、铺垫手法、节奏控制、句式语感、负空间、AI腔黑名单）。

- 输入：完整源文 271824.txt（分批，每批 N 行）
- 流程：分批读 → 每批提炼手法片段 → 归并去重 → 产出手法卡 chunks
- 产出：chunk_情感高潮手法 / chunk_铺垫手法 / chunk_节奏控制手法 / 各场景手法卡 / 负空间 / AI腔黑名单
- 铁律：只提炼手法，绝不产出角色/世界观/设定卡（边界已写进 prompts/analyst.md）
- 频率：每本书一次
- 现状：**提示词已就绪，但 run_pipeline 没有 `--analyst` 入口，没人能跑它**

### 生成器 3：正则统计（analyze.py，已完成）
- 已扩维：句式/段落/感官分布/对话风格/章末钩子/词汇质感指纹
- 已是通用的（只 271824.txt 文件名要参数化）
- 频率：每本书一次

**三个生成器的关系**：analyze.py（正则量化）+ analyst（LLM学手法）→ 喂给 → 设定生成器 + 写手风格卡。

## 五、实施顺序（分三步，每步可独立验证）

### 第一步：抽配置、解耦脚本（地基，风险最低）
1. 新建 `book.config.json`（填入当前这本书的值）
2. 加 `load_book_config()`
3. 把 A 类 9 处硬编码逐个改为读 config
4. 验证：现有 6 章 dry-run，token/行为与改造前一致 → 证明"抽配置没改坏现有书"
5. 再做"换书冒烟测试"：临时塞一个假 config（不同人名/境界），跑 dry-run 不报错 → 证明"换书不死"

### 第二步：补 analyst 入口 + 跑通手法分析
1. run_pipeline 加 `--analyst` 模式（一次性，不进每章循环）
2. 分批读源文、提炼、归并、产出手法卡
3. 接进写手检索表（高潮章调高潮卡、伏笔章调铺垫卡）

### 第三步：做设定生成器
1. 新建 generate_setting.py
2. 从"一句话立意 + 原文类型"生成整套设定 + book.config
3. 这一步做完，才真正实现"换书=换立意，自动出设定"

## 八、下次跑 analyst 时顺手扩产（不为此重跑当前批）

当前这批 analyst（40k/批 × 61 批）只采了**句子/场景级**手法（句式、对话、高潮、铺垫、节奏、章末钩子、负空间），产出三张卡已验证干净可用。但既然"全量通读一次"成本已经付了，**下次跑 analyst（本书重跑或朋友换新书第一次跑）时**，值得在 `analyst_map.md` 里加两个**结构级宏观维度**，顺手多产两张手法卡：

1. **chunk_世界观铺陈手法.md** — 怎么不靠大段说明、而借动作/闲话/物件把世界规则透出来（反信息倾倒）。消费者：beat_planner、定卷纲的职责、未来的 generate_setting.py。
2. **chunk_大事件编排手法.md** — 一卷里怎么布大高潮、小事件怎么垫大事件、几章一个爽点、伏笔隔多久回收。消费者：beat_planner、卷纲职责。

**铁律不变**：只提炼"怎么塑造世界观/怎么编排大事件"的**手法**，绝不提炼源文的世界观内容/具体事件（那是抄设定，是"玄幻写成灵异"的同类灾难）。区别：「这类小说怎么把世界观立起来」✅ vs 「这部小说世界观是什么」❌。

**为什么不现在加**：加维度后前 60 批没采、数据不一致，等于重跑全量——浪费 token 和时间。这两张卡值得做，但属于"下次第一次跑"的增量，不为它推倒已跑批。

3. **chunk_全书结构手法.md** — 源文的全书骨架是怎么定的:卷结构怎么划分、大转折点间隔多少章、修炼升级节奏怎么分配、长线伏笔埋多久才收/中间怎么推进。消费者：`generate_master_outline.py`(生成全书骨架时参考源文的结构节奏,不抄剧情)。

**铁律不变**：只提炼结构手法(怎么布局),绝不提炼源文的具体剧情/人物/设定。区别：「这类小说大转折点一般隔多少章、用什么类型事件」✅ vs 「源文第100章主角打败了谁」❌。

注意：这与"用模型生成原创世界观"是两件事——后者是第三步的 `generate_setting.py`（仿源文塑造风格、写**原创**世界观），本节是"提炼塑造手法供其他职责参考"。两者互补。

## 九、防 LLM 偷懒：位置标签探针

**问题**：让模型读 15 章/5万字提炼手法，它可能**没读透**——不是技术上装不下（5w 占百万上下文仅 5%，几乎不衰减），而是**行为偷懒**：只盯印象最深的一两章（通常是开头或最炸的那章）发挥，其余十几章一笔带过。百万上下文救不了这个，因为问题不在容量。

**当前缓解（被动）**：61 批 × 频率加权归并——真普遍的手法会在几十批里反复出现，单批偷懒会被群体平均掉。这扛住了大部分风险，但抓不到"整批糊套话"的现行。

**下次要加的探针（主动、近零成本）**：
1. `analyst_map.md` 里让每条观察额外标一个**位置标签**：`位置: 前段/中段/后段`（不破坏"禁原句"铁律，只标位置）。
2. 脚本扫每批输出的位置分布：**若某批的观察全挤在"前段"，就是只读了开头的铁证** → 自动标记该批 `<<RETRY>>`、重跑。
3. 阈值：一批至少要有覆盖中段、后段的观察各 ≥1 条，否则判定偷懒。

**为什么对症**：批大小对"均匀用力"几乎无影响（不是容量问题），位置探针直接验证"是否读遍全批"，才是真正的反偷懒手段。

**何时做**：与第二步（analyst 扩维）一起，下次跑 analyst 前加。当前批已跑完且频率加权可信，不重跑。

## 十、精品逼近层的通用化（如何升级 / 如何通用 / 如何写得好）

精品逼近 Phase 1-3 给系统加了一层逼近精品文的机制（盲感官模块、prose 手法卡、主题论辩/自欺/弧内核账本、线索台账、Best-of-N、境界倒退检测）。这层威力大，但是**贴着沈安手写的**。本节专门定义这层怎么变通用——这正是"另一个计划（让系统变通用）"必须吃下的部分。

先把这层拆成三种东西，因为它们的通用化路径完全不同：

| 类别 | 例子 | 本质 | 通用化方式 |
|---|---|---|---|
| **纯手法**（跨书通用） | 主角能动性、Try-Fail、反讽落差、后续与微张力、克制留白、升级代价（去掉盲眼例子后） | 写作通则，任何小说都适用 | **留在代码/卡库，不进 config**。只需把卡里的沈安例子参数化 |
| **机制骨架**（跨书通用） | thematic_stances/arc_core/self_deception/threads/reveal_ledger 的**结构与 merge 逻辑**、Best-of-N、select_chunks 点亮、境界倒退检测 | 账本字段和算法，与具体书无关 | **已经是通用的**。只要境界来源、实体名等从 config 读，机制本身换书即用 |
| **本书内容**（随书走） | 盲感官的"盲"、ledger 里沈安的 arc_core 种子、3 条 thematic_stances 的具体问句、主题意象"盲/见 灯火/灰烬" | 这本书是什么 | **不该手写进系统**，应由生成器产出 / 随设定文档走 |

通用化的总原则一句话：**手法和机制留在系统里（参数化），内容交给生成器和 config**。下面分三问回答。

### 如何升级（让手写层可被覆盖，不返工）
手写卡是"临时占位"，正式版来自 analyst（手法）和 generate_setting（内容）。升级路径已经预埋，只差打通：

1. **手法卡**：手写卡和 analyst 卡**同名**，analyst 版优先（覆盖规则已写进精品计划）。所以升级 = 跑通 §五第二步的 `--analyst` 入口，让 287 万字真实手感的卡自动盖掉手写的通用常识版。**不需要改任何调用方**——select_chunks 按文件名检索，换内容不换名即可。
2. **盲感官模块**：升级成"主角特殊感官模块"。模块标题、主感官清单、禁用词从 `config["protagonist"].sensory_module` 注入；`writer_focus_modules` 默认注入这块的前提改成 `if config 主角 sensory_module.enabled`。沈安盲眼 = 这套通用模块的一个实例。
3. **ledger 种子（arc_core/thematic_stances/self_deception）**：升级成"由 generate_setting 播种"。换书时 runtime 清空，生成器从立意+故事核产出新主角的 want/need/lie/truth 和本书的主题问句，写进新 ledger。系统侧零改动，只是种子来源从"手填"变"生成"。

### 如何通用（换书时这层不串味的检查点）
在 §五第一步的"换书冒烟测试"里**加一项精品层校验**：塞一个假 config（非盲、不同境界、不同主题），跑 dry-run 后断言：
- 主角不盲（`sensory_module.enabled=false`）时，**盲感官模块不进** writer 装配、`check_vision_consistency` 跳过——不会拿盲眼规则误导写手。
- prose 卡里不再出现"沈安/盲"等本书专名（用 §六的占位符替换后，扫一遍卡库零专名）。
- thematic_stances/arc_core 注入读的是新 ledger 种子，不是残留的沈安数据（验证 runtime 清空生效）。
- 4 处境界检测全部用新 config 的境界表（验证 §三的单一来源改造）。

通用化的边界（避免过度抽象）：**纯手法卡不要 config 化**。把"主角能动性""Try-Fail"也塞进 config 是过度工程——它们对任何书都一样，留在卡库即可。只有"内容"和"主角特殊设定"进 config。

### 如何写得好（这层的质量上限怎么提）
通用化解决的是"换书不崩、不串味"，不解决"写得好"。写得好靠三条，且互相独立：
1. **手法卡的真伪**：手写卡是通用常识，**天花板有限**。真正的提升来自 analyst 从同类精品原著学到的"只有这类书才有的手感"。所以"写得好"的最大杠杆是**跑通 analyst 并定期重跑**（配合 §九的反偷懒探针保证读透）——这比任何 prompt 调优回报都高。
2. **Best-of-N**：不换模型下逼近 mimo 能力上限的唯一手段。通用化后，它对任何书都能在关键章开启（打分用免费 code，不耦合本书）。默认关、仅高潮章开的纪律不变。
3. **诚实的天花板**：句子能多漂亮最终仍卡在 mimo 本身。通用化和手法卡只托高下限、压平穿帮和套路；要显著超基线，**换更强的 writer 模型仍是回报最高的单笔投资**（本次按用户决定暂不换，先榨干 mimo）。

### 落地顺序（接在 §五之后，不打断原三步）
精品层通用化**依赖 §五第一步的 config 地基**，所以排在它之后、可与第二步并行：
- **第一步做完后**：顺手把盲感官改成 config 驱动的"主角特殊感官模块"，prose 卡里的沈安例子参数化（B 类债，低风险）。
- **第二步（analyst 入口）做完后**：手法卡自动被真实手感版覆盖——精品层的"手法"部分就正式化了。
- **第三步（generate_setting）做完后**：ledger 种子（arc_core/主题问句/主题意象）由生成器播种——精品层的"内容"部分就正式化了。
- 至此精品逼近层从"沈安专属手写"变成"任何书都能用的通用能力"。
