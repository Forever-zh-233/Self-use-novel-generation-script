# AnalystMap 双段输出校验与重试计划

> 状态：待审核。本文件只记录治本方案，未修改代码或运行产物。

## 结论

Analyst MAP prompt 明确要求输出“手法观察”和“结构台账”两段，且缺一不可；但解析器在缺少“结构台账”标记且没检测到真名特征时，会把整段当手法观察继续处理。这会让结构校准、伏笔/物件/人物跨批拼接静默缺席。

## 当前证据

- `prompts/analyst_map.md:44-45`：要求必须输出两段，并用分隔标记隔开。
- `scripts/run_pipeline.py:1209-1215`：有结构标记时切出 technique 和 structure。
- `scripts/run_pipeline.py:1216-1220`：缺结构标记时，只要没检测到真名特征，就返回 `(technique, "")`，继续跑。

## 根因判断

- prompt 的“严格输出契约”没有在解析层落实为硬校验。
- 解析器优先考虑“防真名泄漏”，但没有处理“结构段缺席就是本批失败”的契约。
- Analyst 的结构台账是后续宏观结构/伏笔/物件来源，缺失不应被当成正常空值。

## 影响

- 结构观察缺批，后续全书结构手法、伏笔间隔、物件复现等归并会失真。
- 模型偶发漏段不会被重试，最终产物看起来完整，实则少了一半职责。
- 如果后续规划依赖结构分析，会以为没有结构规律，而不是“本批没产出”。

## 治本方案

1. 把双段输出改为硬契约。
   - 缺 `=== 手法观察 ===` 或 `=== 结构台账 ===` 直接判定本批 MAP 失败。
   - 失败进入重试，不进入 reduce。
2. 保留防污染降级，但不伪装成功。
   - 有真名特征时仍可把整段归为 structure quarantine。
   - 但必须标记 `map_status=invalid_format` 或 `quarantined`，不能当正常 technique。
3. 增加 Analyst 批次 manifest。
   - 每批记录 `has_technique_section`、`has_structure_section`、`retry_count`、`quarantine_reason`。
4. 增加测试。
   - 缺结构台账标记时，断言触发重试/失败。
   - 缺手法观察标记时，断言触发重试/失败。
   - 含真名且缺标记时，断言进入隔离结构段，不进入手法卡。

## 验收标准

- Analyst MAP 的双段格式由代码强校验。
- 缺结构台账不会静默产出“正常手法观察”。
- reduce 阶段只消费格式有效或明确隔离的批次。
