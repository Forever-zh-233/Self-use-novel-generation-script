# BeatPlanner 场景可演性语义门禁计划

> 状态：待审核。本文件只记录治本方案，未修改代码或运行产物。

## 结论

BeatPlanner prompt 要求冲突具体、章节可执行、爆点有可演动作；但主流程只做格式解析和少量 StoryDirector 方向检查。`normalize_beat()` 还会用“推进主线冲突”“留下下一章问题”等泛化默认值补齐空字段。这样 Writer 可能拿到“格式完整但戏剧上空”的 beat。

## 当前证据

- `prompts/beat_planner.md:41-45`：beat schema 要求填写本章冲突、具体物件、具体动作、信息差、转折。
- `prompts/beat_planner.md:75-80`：要求章节能被写手直接执行，冲突具体到人/物/地点/代价。
- `prompts/beat_planner.md:186-189`：抬档后必须在具体动作里写出爆点动作，并讲清因果和代价。
- `scripts/run_pipeline.py:89-110`：`normalize_beat()` 缺字段时会补 `推进主线冲突。`、`留下下一章问题。`、`小幅兑现一个读者期待。` 等泛化默认值。
- `scripts/run_pipeline.py:138-178`：`beat_direction_check()` 只在 StoryDirector severity>=2 时检查重复模式和钩子型，不检查场景可演性。

## 根因判断

- “可演性/戏剧质量”目前只写在 prompt 里，没有机器或 LLM gate。
- 缺字段默认值让坏 beat 变成“可运行 beat”，掩盖 BeatPlanner 输出质量问题。
- Writer 的上限被 beat 限死：beat 空泛时，Writer 只能扩写空泛冲突。

## 影响

- 未来章节可能稳定生成流水账：有标题、有冲突、有钩子，但没有选择、代价、动作和场景转变。
- Reviewer 事后可以指出问题，但已经浪费 writer/reviewer/editor token。
- 测试只要 JSON 字段存在就绿，抓不到“戏剧上不可执行”。

## 治本方案

1. 增加 Beat 语义质量 gate。
   - 检查 `本章冲突` 是否包含具体角色/目标/阻碍/代价。
   - 检查 `具体动作` 是否至少有一个可拍摄动作，而不是情绪标签。
   - 检查 `转折` 是否改变判断/关系/局面。
   - 检查 `章末钩子` 是否落在具体物件/声音/动作/选择上。
2. 区分字段缺失与合法空值。
   - `normalize_beat()` 不应把核心字段缺失补成泛化内容后放行。
   - 缺核心字段应触发 BeatPlanner 重试。
3. 按张力档位设门槛。
   - `平`：允许小事件，但仍需场景价值变化。
   - `小起伏/小高潮/关键章`：必须有选择、代价、可演爆点动作。
4. 记录 gate 结果。
   - `beat_quality_gate.json` 写出通过项、问题、重试输入。
5. 增加负样本测试。
   - `本章冲突=推进主线冲突`、`章末钩子=留下下一章问题` 必须失败。
   - `小高潮` 但 `具体动作` 为空或抽象，必须失败。
   - 平档日常章若没有任何状态变化，给 warning 或失败。

## 验收标准

- Writer 不再收到格式完整但戏剧上不可执行的 beat。
- BeatPlanner 输出空泛默认值时会被重试，而不是进入正文生成。
- 测试能捕捉“平庸 beat”而非只捕捉坏 JSON。
