# Cleanup 正式卷纲保护与草稿清理边界计划

> 状态：待审核。  
> 范围：只治理清理脚本的权威资产边界，不修改当前卷纲或测试。

## 现网证据

- `scripts/clean_chapter_artifacts.py:2` 文档声称“保留配置/提示词/设定文档/源码/骨架/卷纲”。
- `scripts/clean_chapter_artifacts.py:65-66` 实际删除 `卷纲/10-卷纲.md`。
- `agent.md:105` 写明 `全书骨架/卷纲/chunks` 清除脚本不碰。
- `tests/scenario_test.py:1889` 却断言“卷纲被清”。
- `scripts/pipeline/planning.py:391-399` 在卷纲为空时会触发生成卷纲。

## 根因

系统没有裁定 `卷纲/10-卷纲.md` 是“正式 active contract”还是“可清理 runtime draft”。文档、脚本、测试三者口径相互矛盾：文档说保留，脚本删除，测试固化删除行为。若正式卷纲被清，后续可能在不该重建时触发自动生成，污染规划水位。

## 治本动作

1. 明确卷纲身份。
   - `卷纲/10-卷纲.md` 若是 reviewed active contract，cleanup 不得删除。
   - 自动生成的未审核卷纲草稿必须另放 draft 路径。
2. cleanup 按 manifest 或白名单清理。
   - 只清章节产物、运行草稿、可恢复中间态。
   - 不按“能重跑”粗暴删除正式规划合同。
3. 测试改为验证边界。
   - 正式卷纲保留。
   - 草稿卷纲/下一卷临时稿可清。
   - analyst、chunks、骨架仍保留。
4. 生成卷纲前增加水位判断。
   - 空卷纲不等于需要生成；必须区分“首次缺失”“被误删”“草稿未审核”。

## 验收

- cleanup 后正式 `卷纲/10-卷纲.md` 不被删除。
- 测试不再断言“卷纲被清”，而是断言正式卷纲受保护。
- 自动卷纲生成不会因 cleanup 误删正式合同而触发。
- 清理脚本文档、agent 文档、测试断言三者一致。
