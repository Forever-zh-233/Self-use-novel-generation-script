# POV 写手标题契约与主流程清洗修复计划

## 结论

这是提示词与主流程格式契约冲突：`writer_pov.md` 要求 POV 写手“不要标题”，但主流程最终正文归档要求 `# 第N章 标题`。当前清洗逻辑只会替换已有标题行，不会在缺标题时插入标题。第 85 章 POV 章这次因为模型自行输出了标题而没有爆，但系统正确性依赖模型违背 POV prompt，属于真实管路风险。

## 当前证据

- `prompts/writer_pov.md:59`：POV 写手被要求“不要标题，不要分隔线，不要元信息”。
- `prompts/writer.md:148`：普通写手要求以 `# 第X章 标题` 开头。
- `scripts/run_pipeline.py:628-633`：`视角角色 != 沈安` 时进入 POV 分支。
- `scripts/run_pipeline.py:904-912`：最终清洗只查找已有 `# 第N章` 标题并替换为正确标题；如果正文完全没有标题，`re.sub()` 不会插入新标题。
- `beats/chapter_85.json:4`：第 85 章是韩铮 POV 章。
- `输出/文章/第085章.md:1`：第 85 章实际有标题，说明本次是模型没有遵守“不要标题”，而不是主流程保证了标题。

## 根因判断

- POV prompt 与主流程归档格式没有统一契约。
- 标题归一化函数只处理“标题写错”，不处理“标题缺失”。
- 测试如果只看当前第 85 章产物，会误以为 POV 标题链路正常；真正应该测试的是“POV 写手严格遵守不要标题时主流程是否补标题”。

## 影响

- 未来 POV 章若严格按 prompt 输出无标题正文，最终 `输出/文章/第NNN章.md` 可能无章节标题。
- reviewer / gate / archivist 会看到与普通章格式不同的正文，归档与阅读体验不一致。
- 若后续脚本依赖标题行截取正文，缺标题会影响清洗和追溯。

## 治本方案

1. 统一 POV 输出契约。
   - 方案 A：修改 `writer_pov.md`，要求也以 `# 第X章 标题` 开头。
   - 方案 B：保留 POV prompt “不要标题”，但主流程必须在 POV 输出后无条件补标题。
2. 抽出统一的 `normalize_final_heading(final, chapter, title)`。
   - 有标题：替换为标准标题。
   - 无标题：在正文前插入标准标题。
   - 有思考/元信息：先截取或清洗，再插入标题。
3. 增加 scenario 测试。
   - fake POV writer 返回无标题正文。
   - 断言最终 `writer_final` 与 `输出/文章` 第一行是标准标题。
   - fake 普通 writer 返回错误标题，断言仍能替换。

## 验收标准

- POV prompt 与主流程标题规范不再互相矛盾。
- 无论 writer 是否输出标题，最终正文都稳定以 `# 第N章 标题` 开头。
- 测试覆盖“POV 严格无标题输出”的样本。
