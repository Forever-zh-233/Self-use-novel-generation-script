# Reviewer 复写风险证据契约修复计划

## 结论

Reviewer prompt 要求判断原文复写风险、标志性桥段、语义相似度和图灵盲测，但主流程没有给 Reviewer 源文证据或安全派生诊断；项目纪律又要求不要把源文全文注入角色。这个职责目前没有可执行证据契约。

## 当前证据

- `prompts/reviewer.md:107-118`：要求判断原文独特比喻、标志性台词、专有桥段、语义相似度。
- `prompts/reviewer.md:288-302`：图灵盲测要求抽取原文段落和 AI 正文混评。
- `scripts/pipeline/gates.py:480-494`：Reviewer 输入没有源文片段、相似度报告或派生复写诊断。
- `TESTING.md:70`：测试纪律强调不要把源文全文注入角色链路。

## 根因判断

- “复写风险”从代码硬门禁、派生诊断、LLM 语义评审三种职责混在一个 prompt 里。
- Reviewer 被要求做需要原文对照的判断，但输入只有新正文和少量规则诊断。
- 出于版权/污染风险，不能用“直接塞源文全文”作为修复。

## 影响

- Reviewer 的复写风险分数可能变成主观猜测。
- 维护者以为系统做了源文相似度防线，实际只有有限硬门禁和 prompt 自我约束。
- 如果未来真的把源文片段注入 Reviewer，又可能违反 no-source 纪律并污染生成链路。

## 治本方案

1. 明确复写风险的证据来源。
   - 代码侧：源文专名污染、内部编号泄露、n-gram 命中、禁用短语命中。
   - LLM 侧：只读取安全派生诊断，不读取源文全文。
2. 二选一调整 Reviewer prompt。
   - 方案 A：移除“图灵盲测/抽原文段落”常规职责，只要求读取派生诊断。
   - 方案 B：建立安全抽样器，只给极短派生片段/特征，不给大段源文。
3. `make_review_input()` 增加“复写风险诊断”section。
   - 标明来源、命中数量、命中位置、是否硬阻断。
   - 不包含源文全文。
4. 增加测试。
   - Reviewer prompt 不得承诺未注入的源文材料。
   - Reviewer input 包含复写风险派生诊断。
   - 测试样本不泄露大段源文。

## 验收标准

- Reviewer 的复写风险判断有可见证据。
- 源文全文不进入常规 Reviewer 输入。
- prompt、测试维护指南和主流程输入契约一致。
