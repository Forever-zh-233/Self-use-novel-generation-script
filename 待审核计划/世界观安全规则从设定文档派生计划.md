# 世界观安全规则从设定文档派生计划

> 状态：待审核。本文件只记录治本方案，未修改代码或运行产物。

## 结论

Writer 的世界观安全规则在读取 `02-世界观设定圣经.md` 后，又手工追加了“晏朝、巡夜司、书院、宗门、江湖、荒年、妖祟”等当前书框架。即使换书并替换世界观文档，这段硬规则仍会要求新书贴合旧世界。

## 当前证据

- `scripts/pipeline/context.py:671-677`：`safe_world_bible_for_writer()` 读取世界观设定后追加写手安全规则。
- `scripts/pipeline/context.py:677`：追加文本写死“晏朝、巡夜司、书院、宗门、江湖、荒年、妖祟这些既有框架”。

## 根因判断

- “不要乱发明地名势力”是通用职责；“晏朝/巡夜司/妖祟”是当前书实例。
- 当前实现把通用边界和项目专名写在同一段代码里。
- 该安全规则位于 Writer 基础输入中，优先级高，换书时很难靠下游 prompt 抵消。

## 影响

- 新书世界观被强行拉回当前书专名体系。
- 玄幻、都市、科幻、历史等不同类型会被错误要求出现巡夜司/妖祟等旧框架。
- 世界观设定文档即使正确替换，也会被代码追加规则污染。

## 治本方案

1. 从设定文档或 book profile 派生世界观安全规则。
   - profile 字段：`world_proper_nouns`、`factions`、`locations`、`forbidden_world_terms`、`allowed_new_fact_policy`。
   - 通用代码只渲染这些字段，不写具体专名。
2. 拆分通用规则与项目规则。
   - 通用规则：不要自行发明替换硬设定；新增设定必须有来源；不要说明书式倾倒。
   - 项目规则：本书具体地名、势力、资源名，由 profile 或设定文档抽取。
3. 增加设定文档抽取/校验。
   - 若 profile 未配置专名，可从 `02-世界观设定圣经.md` 提取“地点/势力/资源”等小节。
   - 抽取不到时只注入通用规则，不回退到当前书。
4. 增加换书测试。
   - 假书世界观为都市/科幻，断言 writer 输入不含晏朝/巡夜司/妖祟。
   - 当前书 profile 下仍能渲染当前书专名护栏。

## 验收标准

- `safe_world_bible_for_writer()` 不再硬编码当前书专名。
- Writer 收到的世界观安全规则与当前 book profile/设定文档一致。
- 换书后不会从代码层注入旧世界框架。
