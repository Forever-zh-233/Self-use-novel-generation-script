# 主角感官模块按设定触发计划

> 状态：待审核。本文件只记录治本方案，未修改代码或运行产物。

## 结论

Writer 模块装配层无条件注入“视觉”和“盲感官”，而这两个模块写死当前书“沈安白天近乎全盲、夜里看得清”的特殊设定。换成非盲主角，写手会被错误要求不用视觉；换成盲配角或多视角书，又无法按本章视角角色正确触发。

## 当前证据

- `scripts/pipeline/context.py:1207-1210`：`writer_focus_modules()` 默认每章 `add("视觉")` 和 `add("盲感官")`。
- `prompts/writer_modules/视觉.md:3-7`：模块写死沈安白天视线模糊、强光看不清。
- `prompts/writer_modules/盲感官.md:1-10`：模块标题和规则明确是“沈安怎么看世界”。
- `prompts/writer_pov.md:23-25`：非主角 POV prompt 又默认“不是盲人”，与角色设定驱动相冲突。

## 根因判断

- 特殊感官是角色设定，不是通用写手职责。
- 模块触发目前按“当前书默认主角”注入，而不是按 `本章视角角色`、`出场角色` 和角色 profile。
- Reviewer/consistency 的感官边界也需要同源设定，否则 Writer 写作约束与评审约束会互相打架。

## 影响

- 非盲主角的新书会出现不必要的盲感官限制和错误画面。
- 盲配角 POV 章可能不会注入盲感官，因为系统只认沈安。
- 同一章多视角时，主角/配角感官边界无法区分。

## 治本方案

1. 在 book profile / character profile 定义感官约束。
   - `sensory_profile.enabled`
   - `applies_to`: 角色名或标签
   - `restricted_senses`
   - `substitute_senses`
   - `condition`: 白天/强光/夜晚/受伤等触发条件
   - `forbidden_phrases` 与 `allowed_exceptions`
2. Writer 模块按本章上下文触发。
   - 读取 `beat["视角角色"]`、`出场角色`、场景时间/光线。
   - 只有当前视角角色或关键出场角色命中 profile 时才注入对应模块。
3. 模块内容模板化。
   - 固定 `盲感官.md` 改为通用 `特殊感官.md` 模板。
   - 沈安盲眼规则作为当前书 profile 渲染，不写在通用模块里。
4. 评审与门禁同源。
   - Reviewer 视角感官边界、Consistency 视觉检查、Writer 感官模块都读同一份 sensory profile。
5. 增加测试。
   - 非盲主角 profile：writer 输入不含盲感官规则。
   - 盲主角 profile：主角章注入对应规则。
   - 盲配角 POV：视角切到配角时注入配角规则。

## 验收标准

- 感官限制由角色设定触发，不再无条件绑定沈安。
- Writer、Reviewer、Consistency 对“谁能看见什么”使用同一权威源。
- 换书 smoke 测试能证明非盲主角不被盲感官污染。
