# 全局境界时点一致性校验修复计划

## 结论

这是管路级一致性问题：全书骨架、卷纲、active_arcs、beat、summary、state 都能用自然语言描述“境界/息坎/第四息/横纹”，但系统没有把这些描述归一成可比较的境界节点。结果不是单纯“突破提前或推迟”，而是同一卷内同时出现多套口径：第 93 章像是临界松动，active_arcs 又说叩门已突破，第 99 章卷纲和正文再写一次过横纹，卷末收束却说息坎没过。

## 当前证据

- `全书骨架.md:65,68`：第一卷境界范围是“凡人→叩门”，closing_state 是“踏入叩门境”。
- `全书骨架.md:265`：第一卷能力表写“凡人→叩门”。
- `卷纲/10-卷纲.md:6`：当前卷纲顶部写“凡人末期，叩门境前兆（引气入体突破两息、息坎松动）”。
- `卷纲/10-卷纲.md:38,69,80`：第 99 章又要求“第四息过手腕横纹”，并把 F-225 写成“息坎突破”。
- `卷纲/10-卷纲.md:61`：卷末收束却写“修为在两息附近徘徊，息坎松了但没过”。
- `runtime/active_arcs.json:11`：当前弧线写“叩门境突破已经发生在93章（松手腕丝线前蹭半寸），本弧不再安排修炼突破”。
- `beats/chapter_93.json:7,23,25`：第 93 章 beat 写“第四息坎松动但没破”“息坎松动了”，更像临界而非完整叩门。
- `输出/文章/第093章.md:29,35`：正文写“丝线往掌心方向挪了半寸”“息坎松了”，没有明确写“踏入叩门境”。
- `runtime/summaries/chapter_093.json:41`：summary 又写“沈安在后院修炼突破‘息坎’”，把“松了”升级成“突破”。
- `输出/文章/第099章.md:27-37,67`：第 99 章正文再次写第四息过横纹、热气到肘弯，并触发眼盲封印警告。
- `runtime/summaries/chapter_099.json:55`：summary 写“沈安在镇中修炼突破后感知愿债未消”。
- `runtime/state.json:838` 仍保留“卡在同一个位置——极限还没突破”；同一文件 `runtime/state.json:900,1181` 又写“第四息过了横纹”“走到肘弯”。

## 根因判断

- 系统没有把“叩门境”“息坎”“第四息”“手腕横纹”“热气到肘弯”建成结构化层级，导致 summary/active_arcs 可以把小关卡升级成大境界。
- active_arcs 保存前缺少高层正典一致性校验；它可以把第 93 章解释为“叩门已破”，但 beat/final/state 仍按“松动/临界”继续写。
- 卷纲内部也缺少自检：第 99 章表格写突破，卷末收束又写没过。
- summary 和 Archivist 写入 state 时没有做“事实强度”区分，`松动/前蹭半寸/过横纹/叩门境` 被混用成同级事实。

## 影响

- 第二卷若按骨架从“叩门→通脉”继续，而读者看到的是“第一卷只松动/又反复过小关”，修炼线会断档。
- 后续 beat_planner 可能同时收到“本弧不再突破”和“第 99 章必须突破”的互斥指令。
- summary/state 会把未确认的修炼小关升级成正典境界，后续 FactChecker 又可能把正文正常回落误判成境界倒退。

## 治本方案

1. 建立境界节点结构化模型。
   - `realm_stage`：凡人、叩门、通脉等大境界。
   - `sub_milestone`：两息、第四息、息坎、手腕横纹、肘弯等小关。
   - `status`：not_started、signaled、loosened、crossed、stabilized、failed、retconned。
   - `evidence_strength`：hint、attempt、partial、confirmed、canon_change。
2. 高层正典优先级校验。
   - 全书骨架定义卷末大境界目标。
   - 卷纲可安排小关和突破章，但内部“阶段表/收束/伏笔表”必须自洽。
   - active_arcs 只能细化，不得把小关静默升级成大境界。
3. 保存 active_arcs 前做境界一致性检查。
   - 若 active_arcs 写“第93章已叩门”，但第93章 final/summary/state 只有“息坎松动”，必须要求 `canon_change_evidence` 或降级为小关进展。
   - 若卷纲同一文件同时写“第99章突破”和“卷末没过”，阻断进入 beat_input。
4. summary / Archivist 增加事实强度门禁。
   - “息坎松了”不能自动总结成“突破叩门”。
   - “第四息过横纹”必须归入 sub_milestone，除非正文明确写大境界确认。
5. 增加 scenario 测试。
   - 构造“第93章 beat/final 只写松动，active_arcs 写叩门已破”的样本，断言保存前报 `realm_strength_upgrade`。
   - 构造“卷纲第99章突破，卷末收束没过”的样本，断言 beat_input 前报 `outline_internal_realm_conflict`。
   - 构造 summary 把“松动”写成“突破”的样本，断言 Archivist/state 写入前降级或阻断。

## 验收标准

- 境界大阶段和修炼小关不再混用。
- active_arcs、卷纲、beat、summary、state 的境界字段能做机器可读对账。
- 高层卷末境界目标若要改写，必须生成正典变更提案，而不是在 runtime 弧线里静默漂移。
