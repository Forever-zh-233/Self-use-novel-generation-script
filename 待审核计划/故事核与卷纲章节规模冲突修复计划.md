# 故事核与卷纲章节规模冲突修复计划

## 结论

根设定文档之间存在明确章节规模冲突：`09-故事核.md` 仍写“每站3-8章，第一卷约20章”，但当前卷纲已经是第一卷 1-100 章。这两个文档都会进入规划链路，容易让 planner 对节奏和阶段规模产生互相矛盾的判断。

## 当前证据

- `09-故事核.md:186`：
  - `每站3-8章，第一卷约20章。`
- `卷纲/10-卷纲.md:5`：
  - `章节范围：第1章 - 第100章（约100章）`
- `卷纲/10-卷纲.md:12`：
  - 北砚县城：第1-20章
  - 青石镇：第21-95章
  - 卷末南行：第96-100章

## 影响

- `story_director`、`arc_planner`、`beat_planner` 同时读到“第一卷约20章”和“第一卷约100章”，会对推进速度产生冲突。
- 当前已经生成到第75章，仍在第一卷中段/后段；如果故事核继续说第一卷约20章，会让审计层误判“严重拖长”。
- “每站3-8章”的漂泊式结构与当前青石镇长驻 70+ 章也冲突，需要明确这是旧设定还是新策略。

## 修复建议

1. 明确权威源：
   - 若当前版本采用 100 章第一卷，则更新 `09-故事核.md` 的第一卷方向。
   - 若“每站3-8章”仍是全书原则，则卷纲应拆分青石镇为多个站内子弧，而不是一个长站。
2. 在 `09-故事核.md` 中区分：
   - 全书早期设计旧值。
   - 当前卷纲实际值。
   - 当前以哪个为准。
3. 给 planner 输入增加“当前权威章节范围”说明，避免旧故事核短尺度压过卷纲。
4. 增加 check 测试：
   - 根设定里的“第一卷约 N 章”不得与当前卷纲章节范围冲突。
   - 若保留旧值，必须标注“旧版/废弃/历史备注”，避免被 planner 当硬约束。

## 验收标准

- `09-故事核.md` 与 `卷纲/10-卷纲.md` 在第一卷长度上不再互相冲突。
- planner 输入中只出现一个当前权威章节规模。
- `scripts/run_tests.py check` 能捕捉未来类似冲突。
