# 旧备份可执行入口退役与隔离计划

> 状态：待审核。本文件只记录治本方案，未修改代码或运行产物。

## 结论

`scripts/run_pipeline_backup.py` 是备份文件，但仍是可执行 Python 入口，并保留大量当前书硬编码。误运行会绕过主流程修复，把旧世界观、主角、感官和能力规则重新带回生成流程。

## 当前证据

- `scripts/run_pipeline_backup.py:152`：默认当前地点为“北砚县”。
- `scripts/run_pipeline_backup.py:1267`：安全世界观规则写死“晏朝、巡夜司、书院、宗门、江湖、荒年、妖祟”。
- `scripts/run_pipeline_backup.py:1431`：别名表写死 `沈归舟→沈安`。
- `scripts/run_pipeline_backup.py:1535`：缺出场角色时默认 `["沈安"]`。
- `scripts/run_pipeline_backup.py:3366-3368`：beat normalizer 默认 `"主线"` 和 `["沈安","黑子"]`。

## 根因判断

- 备份文件和可执行入口没有隔离。
- 它不参与新主流程计划和测试，却仍可能被命令行、脚本或维护者调用。
- 备份文件内的旧逻辑会长期成为“旁路污染源”。

## 影响

- 主流程修好后，备份入口仍可生成旧书污染输入。
- 换书时误运行会把当前书主角、地点、世界观注入新书。
- 测试覆盖主入口也无法证明备份入口安全。

## 治本方案

1. 退役可执行备份入口。
   - 移入 archive，或改名为非 `.py` 运行入口。
   - 文件头部加入不可运行 guard。
2. 建入口白名单。
   - 只有白名单入口可执行生成。
   - CI/测试扫描 `scripts/*pipeline*.py` 中的 main/call_role 入口。
3. 备份内容转为版本历史。
   - 真需要参考旧代码时使用 git 历史，不保留活的旁路入口。
4. 增加污染扫描。
   - 退役入口不得被 README、bat、计划任务引用。
5. 增加测试。
   - 运行入口发现器，断言 backup 不在可执行入口列表。

## 验收标准

- 备份文件不能被误当生成入口运行。
- 主流程入口白名单清晰。
- 当前书硬编码不会通过 backup 旁路进入未来生成。
