# 未来章节精品逼近闭环计划

> 状态：待审核；第一批管路修复已落地。
> 本计划只面向未来生成质量，不润色、不补写、不回修已落盘正文；已落盘章节只作为失败模式样本。

> 本轮已落地：`style_metrics` Writer/POV Writer 短摘要消费、POV Writer critical chunk/写作模块接入、叙述区“不是A是B”硬阻断、clean 模式最小审计链留存、提交态正文 hash/committed gate 审计，以及对应 quick/scenario/check 测试。

## 一、目标

建立“未来章节精品逼近闭环”，让系统稳定识别并拦截“无错但平庸、能读但没戏、信息正确但不好看”的章节。每章必须产生一个不可省略的戏剧变化，并由角色选择、场景冲突、信息推进、情绪转折或世界揭示承担；如果一章删掉后不影响人物关系、事实状态、伏笔账本或后续选择，即使没有穿帮，也不能直接放行。

本计划的关键不是“让 Writer 多写好一点”，而是证明 Master / Volume / Arc / Beat / Writer / Reviewer / Editor / FactChecker / FinalGate 每个职责声称的能力都有真实输入、明确判断、结构化输出、下游消费、质量门禁和测试覆盖。

## 三、硬边界

- 不修改已落盘正文；已落盘章节只用于反向诊断未来会复发的系统失败模式。
- 不把当前书具体剧情、人物、地点、章节安排硬编码进通用 prompt 或通用代码。
- 不靠疯狂加 prompt 解决质量问题；所有注入必须有 token 上限、权威来源、消费端验证和缺失报警。
- 不靠强制截断伪装省 token；截断只能作为最后降级保护，默认路径必须是按需检索、按职责摘要、按优先级压缩。
- 不为了质量门禁牺牲通宵挂机；默认策略是标记、隔离、复核队列和可续跑，只有显式严格模式才停机。
- 不把测试绿当完成；测试必须证明主流程真实消费和门禁真实生效。
- 大报告不得直接塞给角色；Analyst 产物必须先资产化、审核化、摘要化、按职责注入。

## 四、失败模式标签

把已落盘章节、beat、弧线、章纲、summary、review 记录作为样本库，抽象出通用负样本标签：

- `无选择`：角色没有做出会改变局面的选择。
- `假冲突`：看似有危险或争执，但已有能力/常识可以轻易绕过。
- `节奏空转`：连续章节停在同一种事件模式，没有阶段功能差异。
- `信息重复`：只是换话术提醒旧线索，没有新增信息或新解释。
- `角色工具人`：配角只递线索、夸主角、制造障碍，没有自己的目的。
- `世界平面化`：连续多章只处理眼前事务，没有远方势力、高层规则、历史遗留或未知现象的影子。
- `情绪无递进`：角色状态描述很多，但情绪没有因选择和后果发生变化。
- `伏笔假推进`：伏笔只是被提起，没有变形、加压、部分兑现或制造新问题。
- `场景不可见`：缺少可见动作、空间调度和感官落点，只有解释。
- `对话传声筒`：对话只传递信息，没有潜台词、遮掩、试探或立场。
- `后果不回写`：事件发生后没有改变 ledger/state/summary/下一章规划。
- `概括腔/总结腔`：段落或章尾替读者总结主题、情绪或意义。
- `禁令失效型AI腔`：prompt 明令禁止但模型仍稳定复发的句式，例如“不是A是B”。这类不再靠提醒解决，必须进确定性扫描、Reviewer blocker 和 Editor 定向改写。
- `钩子失约`：前章章末钩子已经埋出方向，但后续 1-3 章没有回应、升级、转义或兑现。
- `机制欠解释`：系统/规则给出判定结果，却不给角色和读者可行动的判断依据；不是“暂时不揭真相”，而是缺少下一步推理入口。
- `压力线断供`：倒计时、追杀、清理指令、危机约束在换地点或换段落后消失，导致行走和探索变成等待。
- `金手指仪表盘化`：系统只显示状态，不再改变选择、能力、代价、局势或后续任务。

## 六、落盘病灶专项治理

1. AI 腔禁令失效治理。
   - “不是A是B”这类稳定复发句式不再只写在 Writer prompt 里。
   - gates 必须做确定性扫描；Reviewer 必须把命中视为 blocker；Editor 必须收到原句和替换目标，做局部改写。
   - 验收不是“prompt 里有禁令”，而是连续生成章节不再全章命中，命中时不会通过 FinalGate。

2. 钩子-回应链治理。
   - 每个章末钩子进入短期 hook ledger，记录 `hook_id / planted_chapter / expected_response_window / required_response_type`。
   - BeatPlanner 必须在窗口内选择回应方式：兑现、升级、转义、误导、延迟并给新证据。
   - Ch70-Ch72 这类钩子后接 Ch74-Ch76 连续蓄势但行动延迟的模式，必须被标记为 `钩子失约风险 + 节奏倒退风险`；Ch73 这类有实质信息增量的章节不得误伤。

3. 可删除弱章治理。
   - 对连续 5/10/20 章做“删章影响”检查：删掉本章是否影响关系、事实、伏笔、能力、地点、压力线或下一步选择。
   - Ch98-Ch100 这类氛围强但主线弱连接的段落，不是禁止写，而是必须绑定至少一条主线后果：窑厂线、修炼线、系统机制、角色选择或世界纵深。

4. 机制规则可行动性治理。
   - 系统判定可以保持神秘，但必须给角色一个可行动的推理入口。
   - “了结方式不符”不能只显示失败结果；至少要给出冲突维度，例如对象错、方式错、愿望理解错、代价未付、执念未触及。
   - 验收：读者不知道终极答案可以，但必须知道沈安下一步为什么这样试。

5. 压力线连续性治理。
   - 倒计时、清理指令、追索、危机约束必须跨场景延续。
   - 换地点后，BeatPlanner 输入必须带“当前压力线还剩什么、下一章如何继续咬住”。
   - Ch101-Ch107 雾南泽这类路途场景必须每章推进至少一项：压力逼近、线索缩小、能力变化、关系改变、机制理解。

6. 系统故事引擎治理。
   - 系统不能长期只做状态显示。
   - 每个系统事件必须归入一种功能：触发新选择、开放/限制能力、改变治疗方式、制造代价、引出任务、揭示机制、推动关系。
   - 若连续若干章系统只显示方向/等级/债务而不改变叙事，BeatPlanner 和 Reviewer 必须报警。

## 七、Analyst 产物资产化计划

1. 建立 Analyst 产物清单。
   - `runtime/analyst/map_*.md`：批次观察，原则上不直接消费。
   - `runtime/analyst/_reduce_output.md`：手法卡生成中间稿，不直接消费。
   - `runtime/analyst/_structure_calibration.md`：结构校准报告，只作为生成待审核配置的来源。
   - `config/structure_norms.json`：结构节奏参考分布，可按 scope 注入规划层。
   - `分析草稿/style_metrics.json`：句长、段落、对话、章末钩子等风格指标，应转为 Writer / Reviewer 可消费短摘要。
   - `chunks/index.json` 与 `chunk_*.md`：手法卡和旧资产混合目录，必须按 category / consumer_scope 白名单消费。

2. 建立消费合同。
   - 每个分析资产必须有 `source / review_status / consumer_roles / scope / max_tokens / stale_policy`。
   - 没有消费者的资产不能假装生效；要么退役，要么挂入明确职责。
   - 任何 prompt 声称“参考原文手法/节奏/风格”，对应输入必须有 section；否则测试失败。

3. 建立摘要器。
   - `structure_norms` 继续按 `volume / arc / beat` scope 输出。
   - `style_metrics` 只输出 Writer 能执行的短摘要，例如句长范围、段落节奏、单句成段使用时机、对话动作尾巴比例、章末钩子倾向；禁止输出源文专名和原句。
   - craft chunks 按职责分层：规划 craft、写作 craft、情感 craft、场景 craft、角色/正典资料分离。

## 八、落盘章节反向诊断计划

1. 抽样窗口。
   - 连续 5 章：检查直接重复、节奏空转、信息增量。
   - 连续 10 章：检查角色存在感、伏笔推进、世界纵深。
   - 连续 20 章：检查卷级张力曲线、模式疲劳、阶段功能。

2. 每章质量损耗追踪。
   - Beat 是否已经不可演。
   - Writer 是否把冲突写成任务清单或解释文。
   - Reviewer 是否放过平庸稿。
   - Editor 是否只润色而没有修结构。
   - FinalGate 是否只拦穿帮不拦无戏。
   - Summary / ledger 是否把计划态误当事实态回灌。

3. 章节可删除性测试。
   - 删除本章后，如果人物关系、事实状态、伏笔账本、后续选择都不受影响，则标记为弱章样本。
   - 弱章样本不回修正文，只归纳成负样本，反向补 BeatPlanner / Reviewer / FinalGate。
   - 对 Ch74-Ch76、Ch98-Ch100、Ch101-Ch107 这类已知样本，要分别归入连续蓄势不行动、孤立氛围段、压力线断供三类测试夹具；Ch73 作为“有信息增量但后续承接不足”的对照样本。

## 九、测试与门禁

1. 消费端快照测试。
   - VolumePlanner 输入必须包含卷级结构参考。
   - ArcPlanner 输入必须包含弧长、节点间距、伏笔窗口等结构参考。
   - BeatPlanner 输入必须包含章级节奏、近期事实、当前能力和 active arc 走向。
   - Writer 输入必须包含按需手法卡；若存在已审核 `style_metrics`，必须包含短风格执行摘要。
   - POV Writer 输入必须包含与主角 Writer 等价的 critical 约束：`黄金法则 / 负空间 / AI腔黑名单 / 本章写作要点`，并明确哪些手法卡允许按需注入。

2. 未消费资产测试。
   - 扫描 `chunks/index.json`、`config/structure_norms.json`、`分析草稿/style_metrics.json`、`runtime/analyst`，生成“有资产但无消费者”的报告。
   - 对没有消费者的资产，必须标记 `retired / draft_only / needs_consumer / fixture_only`，不得静默存在。

3. 审查证据留存测试。
   - 每章至少保留 Reviewer 输入、Reviewer 原文输出、结构化 verdict、初稿 gate、终稿 gate、Editor 输入、Editor 输出、final_gate 复检摘要；清理模式也不得删除诊断链，只能压缩或归档。
   - 分数表只能做早晨速览，不能替代可复盘证据；如果某章“是否返工：是”，必须能追到具体 blocker、原句、修稿动作和复检结果。
   - 续跑时复用旧评审或旧修稿，必须在证据里标注来源，避免把上一次失败结论误当成本轮已修复。

4. 负样本测试。
   - 构造干净但无戏、假冲突、信息重复、工具人配角、世界平面化、AI 总结腔、伏笔假推进等样本。
   - Reviewer / FinalGate 必须给 blocker，不得只靠总分放行。
   - 增加落盘症状负样本：全章“不是A是B”、三章重复退回、章末钩子失约、机制判定欠解释、路途场景压力线消失、系统仪表盘化。

5. Token 门禁。
   - 每个分析资产摘要必须有 max_tokens。
   - 超预算时按职责降级，不允许整份 `_reduce_output.md`、`_structure_calibration.md` 或 `style_metrics.json` 直接注入。
   - 验收必须区分“按需省 token”和“硬截断丢信息”：critical 信息不得被截断，high/normal 信息必须先摘要或检索，最后才允许降级跳过。
   - 通宵挂机模式下，质量问题默认进入待复核队列并继续后续章节；不能因为单次 Reviewer/FinalGate 判断抖动导致整晚停机。

## 十、实施顺序

1. 先补审查证据留存：保证每章能复盘 Reviewer / Editor / FinalGate 的输入、输出、阻断和复检结果。
2. 处理确定性复发且证据最强的 AI 腔禁令失效：扫描、Reviewer blocker、Editor 定向改写、FinalGate 阻断。
3. 统一主角 Writer 与 POV Writer 的 critical 约束和手法卡消费合同，避免非主角视角绕过精品管路。
4. 再建钩子-回应 ledger 和压力线连续性门禁，防 Ch74-Ch76、Ch101-Ch107 这类原地打转。
5. 再做可删除弱章检测和落盘章节样本诊断，防 Ch98-Ch100 这类孤立氛围段。
6. 再治理系统故事引擎退化，确保系统事件不只是显示，而是改变选择、能力、代价或局势。
7. 再建资产消费矩阵和未消费资产扫描，补 `style_metrics` 的 Writer 短摘要消费合同。
8. 最后把测试并入主流程场景测试，防止以后新增手法卡、提示词或分析产物又写了没人吃。

## 十一、完成标准

- 每个核心职责都有“声称能力 -> 所需输入 -> 实际注入 -> prompt 消费 -> 输出字段 -> 下游消费 -> 门禁 -> 测试”的闭环证据。
- 所有 Analyst 产物和手法卡都有消费者、退役状态或待审核状态；不存在“写了但没人吃”的隐形资产。
- 主角 Writer 与 POV Writer 都能吃到 critical 约束和必要手法卡；不存在某条写作分支绕过 AI 腔黑名单或精品手法卡。
- 每章评审证据可复盘：只有分数表而没有 reviewer/gate/editor 诊断链的状态不再视为合格留存。
- 未来新章即使没有事实错误，只要没有真实戏剧变化、没有信息增量、没有角色选择、没有压力线延续或没有精品潜力，也不能通过主流程。
- “不是A是B”这类确定性 AI 腔不再出现全章全命中、全书全命中；一旦命中，不能绕过 Reviewer / Editor / FinalGate。
- 章末钩子、压力线、系统机制、系统奖励/代价都有后续承接；不会再出现连续多章气氛成立但主线无痕的段落。
- token 成本受控：所有分析资产和历史信息都按需注入，不全盘塞入；信息降级有记录，不能用静默截断造成关键事实缺失。
- 通宵挂机可持续：质量门禁发现问题时能标记、隔离、续跑和复核，不把自动执行变成频繁人工守夜。
- 已落盘章节暴露出的失败模式会被提前识别、阻断、返工或反馈给下一章规划，而不是靠人工读完后再抱怨“怪怪的”。
