# 核心配角标注链路补齐计划

## 结论

Volume Planner 要求为“标注为核心配角”的新登场角色规划整章 POV，但上游 master_outline 的 `new_characters` 只是字符串列表，没有“核心配角/路人/一次性配角”的结构化标注。下游职责依赖一个不存在的上游字段，导致核心配角 POV 规划只能靠模型临场猜。

## 当前证据

- `prompts/volume_planner.md:80-81`：要求对本卷新登场角色中标注为“核心配角”的角色规划整章 POV 章位置。
- `prompts/volume_planner.md:88`：要求新角色与全书骨架 `new_characters` 对得上。
- `prompts/master_outline.md:42`：`new_characters` schema 为字符串列表 `["本卷登场的重要角色"]`，没有角色级类型、重要度、是否核心配角、是否跨卷存在等字段。
- `scripts/pipeline/planning.py:602-619`：run_volume_planner 输入由全书骨架、当前状态、ledger、期待账本等组成，没有对 `new_characters` 补出核心配角分类。

## 根因判断

- 全书骨架、卷纲规划和 POV 规划之间缺少角色重要度 schema。
- “核心配角”是下游硬职责，但没有上游数据来源，也没有校验器检查卷纲是否兑现。
- 这会让模型在卷纲阶段自己发明“谁是核心配角”，与故事核或长期角色规划漂移。

## 影响

- 重要配角可能缺少 POV 暗线渠道，导致后面临时加视角或靠旁白补解释。
- 路人角色可能被误判为核心配角，占用整章 POV。
- 弧线规划师和章节规划师无法稳定知道哪些配角应承担跨卷暗线。

## 治本方案

1. 扩展 master_outline 的 `new_characters` schema。
   - 从字符串列表改为对象列表。
   - 字段建议：`name`、`role_type`、`importance`、`core_supporting`、`cross_volume`、`pov_candidate_reason`、`first_volume_task`。
2. Volume Planner 只依据该结构化标注规划核心配角 POV。
   - 未标 `core_supporting=true` 的角色默认不规划整章 POV。
   - 如需升级角色重要度，必须在输出中说明理由并回写待审核建议，而不是静默改。
3. 弧线规划和 beat 输入接入该角色分类。
   - 核心配角：可获得暗线/POV 规划。
   - 一次性角色：只允许章内短切或普通出场。
4. 增加结构测试和 scenario 测试。
   - master_outline prompt schema 必含角色对象字段。
   - volume_planner input 中能看到核心配角标注。
   - 卷纲输出若规划未标注角色 POV，应被校验提醒。

## 验收标准

- “核心配角”不再是下游凭空判断。
- 卷纲 POV 规划有稳定上游标注。
- 测试覆盖 master_outline → volume_planner 的角色标注链路。
