# 根目录设定文档归属重构计划

## 目标
把当前平铺在工作台根目录（`E:\Novel 1\`）的一堆核心设定文档归入子目录，让根目录整洁。**纯目录归属调整，不改任何文档内容。**

## 风险等级：高
这些编号设定文档是管线的命门，被脚本、prompt、bat、README 大量引用（脚本内联 77 处、prompt 6 处）。**漏改任何一处引用，管线就会因为读不到文件而崩溃。** 必须用"集中常量 + 全量替换 + 跑测试验证"的方式做，禁止手工逐个猜。

## 三、执行顺序（严格按此序，每步可回滚）

1. **建目录**：`设定/`、`docs/`（B 组若单独建 `账本/` 则一并建）
2. **移文件**：用 `git mv` 移动 A/C 组（保留 git 历史）。B 组按决策移动。
3. **改 core.py**：新增 `SETTINGS_DIR` 和所有具名常量，改 `MASTER_OUTLINE_FILE`/`LONG_FORESHADOWING_FILE` 路径
4. **改各模块内联引用**：planning/gates/context/archivist，全部换成 core 常量（含顶部 import）
5. **改独立脚本**：split_docs.py、clean_chapter_artifacts.py
6. **改 prompt 文本**：archivist.md、beat_planner.md 的路径字符串
7. **改文档**：README-使用说明.md 等
8. **处理 .gitignore**（如需）
9. **跑测试**：`python scripts/run_tests.py all` —— 必须 0 失败
10. **跑一轮 dry-run 或真实第1章**：确认管线能读到所有设定文件，不报 FileNotFoundError

## 六、关键提醒
- 全程用 `git mv`，不要用普通 mv，保留历史。
- 改完**必跑测试**（这是本项目铁律）。测试用的是临时工作台（isolated_workspace），不依赖真实根目录文件，所以测试能验证常量引用是否自洽，但**验证不了真实文件是否在新位置**——所以第 10 步真实跑一章不可省。
- 不改任何文档的**内容**，只改**位置和指向它们的路径**。
- 安全约束照旧：不碰 `.env.local` 明文 key；分数表（输出/分数表/）依然 write-only 不注入。
