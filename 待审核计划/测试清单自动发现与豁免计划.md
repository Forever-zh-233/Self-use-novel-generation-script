# 测试清单自动发现与豁免计划

## 结论

当前 `check` 测试依赖手写清单维护核心文件、固定 prompt 和角色配置。清单已经多次落后于真实入口，说明问题不只是漏加某个文件，而是测试体系缺少“自动发现真实入口 + 显式豁免”的机制。

## 当前证据

- `tests/checks_test.py:15-20` 的 `PYTHON_GLOBS` 未覆盖 `scripts/consistency/*.py`。
- `scripts/consistency/scan.py:21`、`scripts/consistency/mapper.py:22`：consistency 是真实 CLI/LLM 管路，不是死代码。
- `tests/checks_test.py:51-64` 的 `REQUIRED_PROMPTS` 只维护根目录 prompt 清单。
- `scripts/consistency/mapper.py:19`、`scripts/consistency/mapper.py:64`：consistency 使用独立 prompt 目录 `scripts/consistency/prompts/map_agent.md`。
- `scripts/pipeline/summarizer.py:18`：summarizer prompt 是内联常量，不会被文件清单发现。
- `测试文件维护指南.md:43`、`测试文件维护指南.md:104`、`测试文件维护指南.md:180-190` 仍要求人工把新核心模块/prompt 加入固定清单，但没有要求自动发现或豁免说明。

## 根因判断

- `check` 只证明“清单里的东西存在”，不证明“真实入口都在清单里”。
- 非根目录 prompt、内联 prompt、子工具 CLI、固定 role 常量没有统一登记表。
- 测试维护指南仍按旧的手动清单思路写，无法防止下一次新增入口漏测。

## 治本方案

1. 增加入口自动发现。
   - 扫描 `PROMPTS_DIR / "xxx.md"`、独立 prompt 目录、内联 prompt 常量、固定 LLM role 常量。
   - 扫描 `scripts/**.py`，至少对所有非豁免脚本做语法检查。
2. 增加显式豁免表。
   - 对 legacy prompt、一次性脚本、非运行入口必须写明豁免原因。
   - 豁免表本身纳入测试，避免“沉默漏测”。
3. 将发现结果与 `REQUIRED_FILES`、`REQUIRED_PROMPTS`、角色配置清单对账。
4. 更新 `测试文件维护指南.md`。
   - 新增 consistency 子工具、manifest/audit/trace、time anchor、summary fact strength 的维护决策行。
   - 明确“新增入口优先自动发现；无法发现才加入豁免表”。

## 验收标准

- 新增真实 prompt/role/核心脚本时，不登记就会被 `check` 发现。
- 非标准入口有清晰豁免原因，不再靠维护者记忆。
- 测试维护指南与真实测试边界同步。
