# 第二卷卷纲章节边界 101 错位修复计划

## 结论

全书骨架把第二卷定义为 `101-200`，正文、beat、StoryDirector 都已经把第 101 章视为第二卷开篇；但正式卷纲 `卷纲/10-卷纲.md` 把第二卷范围写成 `102-200`，阶段表也从 `102-115` 开始。结果是第 101 章成为未被当前卷纲覆盖的跨卷孤章，下游只能靠标题和事后批注解释它的身份。

## 当前证据

- `全书骨架.md:84-87`：第二卷 `chapter_range: [101, 200]`。
- `卷纲/10-卷纲.md:1-5`：正式卷纲为“第二卷：雾南蛊事”，但章节范围写 `第102章 - 第200章`。
- `beats/chapter_101.json:2-7`：第 101 章标题为“第二卷·开篇”，戏剧目的也写“沈安进入第二卷的第一章”。
- `输出/文章/第101章.md:1`：正文标题是 `第101章 第二卷·开篇`。
- `输出/章纲/第104章_beat_input.md:24-31`：StoryDirector 在第 104 章输入中解释“第101章是第二卷开篇”，但同时把第二卷起段定位为 `102-130`。
- `输出/章纲/第104章_beat_input.md:385-389`、`输出/章纲/第104章_beat_input.md:437-439`：同一 BeatPlanner 输入注入的正式卷纲范围仍是 `102-200`，阶段表第一段为 `102-115`。

## 根因判断

- 骨架、VolumePlanner 输出、正式卷纲、StoryDirector 批注、beat/正文标题之间没有统一的章节范围对账。
- 卷纲生成/提交时只校验“看起来像第二卷”，没有校验其 `chapter_range` 是否与全书骨架目标卷一致。
- StoryDirector 能事后解释第 101 章为“过渡”，但这个解释没有结构化写回卷纲范围，也没有让 BeatPlanner 拿到“101 是第二卷过渡章”的明确元数据。
- 阶段表从 102 开始意味着第 101 章没有卷内阶段职责，后续检查“卷纲兑现度”时容易把它排除在外。

## 影响

- 第 101 章既不在旧第一卷范围内，也不在当前第二卷范围内，成为章节生命周期和卷纲兑现追踪的盲点。
- 后续 BeatPlanner/Reviewer/StoryDirector 会同时看到“101 是第二卷开篇”和“第二卷从 102 开始”，形成隐性自相矛盾。
- 如果以后按卷纲阶段统计节奏、伏笔、核心事件，第 101 章可能被漏算。
- 清理或回滚到卷边界时，系统无法判断第 101 章应随第一卷还是第二卷处理。

## 治本方案

1. 建立卷范围权威源。
   - 全书骨架的 `chapter_range` 是目标卷规划的上游约束。
   - VolumePlanner 输出必须与目标卷 `chapter_range` 对齐，或显式声明 `transition_chapter`。
2. 卷纲提交前做范围对账。
   - `target_volume`
   - `target_chapter_range`
   - `output_chapter_range`
   - `stage_ranges`
   - `transition_chapters`
3. 禁止未声明孤章。
   - 若第二卷范围为 `102-200`，但第 101 章已被 beat/正文/StoryDirector 标记为第二卷，提交应失败或要求补写 `第101章` 的过渡章元数据。
4. StoryDirector 批注不得替代卷纲元数据。
   - “第101章是第二卷开篇/过渡”必须成为可机器读取的范围声明，而不是只存在于 markdown reason。
5. 卷纲兑现追踪要覆盖过渡章。
   - 若存在 `transition_chapter=101`，它也必须有阶段职责、核心事件、主题任务和验收条件。

## 测试要求

- 构造骨架第二卷 `[101,200]`、VolumePlanner 输出 `[102,200]`，断言拒绝覆盖正式卷纲。
- 构造输出 `[102,200]` 但带合法 `transition_chapter=101` 和阶段职责，断言可提交。
- 构造第 101 章 beat/正文标题为第二卷开篇，而正式卷纲不覆盖 101，断言主流程在下一章 BeatPlanner 前报范围冲突。
- StoryDirector reason 中提到“101 是第二卷开篇”但卷纲无元数据时，不得视为已修复。

## 验收标准

- 骨架、正式卷纲、阶段表、beat/正文卷身份不再互相矛盾。
- 每个已生成章节都能归属到唯一卷，或有显式过渡章元数据。
- 卷纲提交失败不会让下游用标题自行拼接卷身份。
