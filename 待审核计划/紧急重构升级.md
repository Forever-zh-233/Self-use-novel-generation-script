# 紧急重构升级计划

## 核心问题

当前 writer.md 有 230+ 行规则，模型在试图同时遵守所有规则，结果：
1. 注意力分散 → 每条规则都"沾一点"但没有一条做到位
2. 规则互相打架 → "要克制"vs"要有情绪裂缝"vs"不要为了提起而提起"
3. 模板化 → 模型把规则当 KPI 完成，写出"正确但无灵魂"的文字
4. Token 浪费 → 大量规则每章都注入，但大部分跟本章无关

## 设计哲学

**精品不是靠规则堆出来的，是靠少数核心原则内化后自然流出的。**

好作者脑子里只有2-3条原则，其余靠直觉。我们要模拟这个：给模型极少的"宪法级"原则让它内化，细节规则只在需要时才出现。

## 重构方案

### 第一层：宪法（永远注入，不超过 300 tokens）

writer.md 精简为只有 3 条铁律 + 基本格式要求：

```markdown
# 写手核心原则

你是写手。根据 beat 和上下文写出本章正文。

## 三条铁律（写每一句话时都要过脑子）

1. **每句话至少做两件事。** 一句话只推进剧情=流水账。好的一句话同时在：推进事件+揭示角色性格，或建立氛围+埋伏笔，或传递情绪+提供信息。如果一句话只做了一件事，问自己：能不能让它顺便再做一件？

2. **具体到只属于这个瞬间。** 如果一个描写换到别的场景也成立（"他皱了皱眉""空气很安静"），那它就是废话。好的描写只能存在于此时此地此人——换个场景就不对了。

3. **能删就删。** 写完一段，假装删掉它。如果读者不会漏掉任何信息或感受，那它就不该存在。精品的标准是"每句话都不能删"。

## 输出格式
# 第{N}章 {标题}
（正文 2500-3500 字）

严禁在正文前输出思考过程。
```

### 第二层：场景工具包（按场景类型注入，每次只注入1个，~400 tokens）

把现有的句式规则、对话规则、节奏规则等**按场景类型重组**为工具包：

- `chunk_工具包_打斗.md`：打斗场景的句式节奏+动作写法+节奏模板
- `chunk_工具包_对话.md`：对话规则+潜台词技法+信息差要求
- `chunk_工具包_日常.md`：日常场景的节奏+细节密度+幽默规则
- `chunk_工具包_悬疑探查.md`：线索铺设+氛围营造+信息释放节奏
- `chunk_工具包_情绪爆发.md`：情绪裂缝+克制写法+节奏突变

每个工具包 300-400 tokens，只在对应场景类型时注入。这样模型每次只看到跟本章相关的规则。

### 第三层：禁忌清单（永远注入，但极短，~150 tokens）

从现有 AI腔黑名单 + 禁止事项中提炼最致命的 5-7 条：

```markdown
## 绝对禁止（触发即重写）
1. 情绪总结（"他感到一阵悲伤"）
2. 解释性对话（角色互相说明已知信息）
3. 章末复述（把本章线索列一遍）
4. 签名动作泛滥（同一动作一章超过2次）
5. 空话过渡（"新的一天开始了""事情还没完"）
6. 抽象描写（"空气中弥漫着紧张"）
7. 主语单一（超过25%句子以"沈安"开头）
```

### 第四层：beat 驱动的深度提示（按 beat 字段触发，0-200 tokens）

只在 beat 标注了对应字段时才注入一句提示：

- beat 有"潜台词机会" → 注入一句："本章对话场景：{谁}在隐瞒{什么}，用行动和岔开话题表达，不直说。"
- beat 有"情绪裂缝" → 注入一句："本章情绪高点在{位置}，沈安有一个压不住的瞬间：{具体反应}。只用一句话。"
- beat 有"内在转变" → 注入一句："本章{角色}从{A}变到{B}，通过一个具体瞬间触发。"

不注入整个 chunk，只注入一句话级别的指令。模型不需要知道"什么是潜台词"的理论——它需要知道的是"这章谁在隐瞒什么"。

### 第五层：角色声音卡（按出场角色注入，每人 ~100 tokens）

现有的角色声音表已经有了，但需要升级——加入"思维方式"和"独特反应模式"：

```markdown
【沈安】
说话：短句，不解释，先做后说。问句比陈述句多。
思维：先算账（值不值），再决定（帮不帮）。犹豫时摸竹杖。
独特反应：遇到危险不是害怕而是"算"——算距离、算退路、算代价。
禁止：不会说教、不会感慨人生、不会主动解释自己的动机。
```

这比现在的"声音=说话方式"更立体——加了思维模式和反应模式，让角色在非对话场景也有辨识度。

## 实施步骤

### Step 1：精简 writer.md
- 当前 230 行 → 目标 30 行（3条铁律 + 格式 + 禁忌清单）
- 所有细节规则移出，不删除，重组为工具包 chunk

### Step 2：创建场景工具包 chunk（5个）
- 从现有 writer.md 的句式/对话/叙事/深度规则中提取
- 按场景类型重组
- 每个 300-400 tokens

### Step 3：改造 select_chunks() 逻辑
- 现有的"必读 chunk"从3个（黄金法则+负空间+AI腔黑名单）改为1个（精简版禁忌清单）
- 场景工具包根据 beat 的"场景类型"字段自动选择
- 深度提示根据 beat 字段是否为"无"来决定注入

### Step 4：升级角色声音卡
- 给沈安、黑子、方绾等核心角色加"思维方式"和"独特反应模式"
- 控制每人 100 tokens 以内

### Step 5：改造 build_writer_sections()
- 移除现有的大段 writer_prompt 注入
- 改为：宪法(300) + 禁忌(150) + 场景工具包(400) + 角色卡(200) + beat驱动提示(0-200) + 状态/账本(已有)
- 总计：~1250 tokens 规则 + ~2000 tokens 状态数据 = ~3250 tokens
- 对比现在：writer.md 本身就 ~3000 tokens + chunks ~2000 = ~5000 tokens
- **净省 ~1750 tokens，同时注意力更集中**

### Step 6：验证
- 跑3章，对比重构前后的质量
- 检查：功能密度是否提升、模板感是否降低、角色辨识度是否提升
- 检查：是否出现"忘了某条规则"的退化（如果有，把那条规则提升到禁忌清单）

## Token 预算对比

| 项目 | 重构前 | 重构后 |
|------|--------|--------|
| writer prompt 规则 | ~3000 tokens | ~450 tokens（宪法+禁忌） |
| 必读 chunks | ~2000 tokens（3个） | ~400 tokens（1个场景工具包） |
| 角色卡 | ~500 tokens | ~200 tokens（精简版） |
| beat 驱动提示 | 0（全靠 writer 自己判断） | ~100 tokens（精准指令） |
| 状态/账本/beat | ~2500 tokens（不变） | ~2500 tokens（不变） |
| **总计** | **~8000 tokens** | **~3650 tokens** |

省出来的 4000+ tokens 可以给正文更多的生成空间，或者让状态数据更完整。

## 防止"照抄模板"的机制

重构后模型看到的不是"规则清单"而是"原则+具体指令"。区别：

**规则清单（现在）**：模型逐条打勾 → 模板化
```
- 短句为主
- 对话要口语化
- 情绪通过动作传递
- 每场景2-3个细节
- 章末要有钩子
...（50条）
```

**原则+指令（重构后）**：模型理解意图 → 自然写作
```
铁律：每句话至少做两件事。
本章指令：沈安在试探女人是否知道朱砂的来源。用行动和岔开话题表达，不直说。
```

模型不需要记住"对话要有信息差"这条规则——它只需要知道"这章沈安在试探，女人在隐瞒"，自然就会写出有信息差的对话。

## 风险与对策

| 风险 | 对策 |
|------|------|
| 规则太少，模型退化成纯 AI 腔 | 禁忌清单兜底（触发即重写） |
| 场景工具包选错了 | beat 的场景类型字段已经很准确，选错概率低 |
| 角色声音卡太短，角色漂移 | gate 检测 + reviewer 评分维度保留"角色辨识度" |
| 金句/功能密度靠原则写不出来 | 在黄金法则 chunk 里加 2-3 个好例子（不是规则，是示范） |

## 需要同步精简的其他职责

### archivist.md（425行 → 目标 150行）

问题：字段太多，模型顾此失彼，经常漏字段或输出格式错误。

方案：
- 把字段分为"每章必填"和"有变化才填"两组
- 每章必填（5个）：latest_chapter、timeline_update、inventory_update（如有变化）、canon.update_entities、recent_events
- 有变化才填（8个）：new_entities、liaoYuan_event、motifs_update、travel_update、factions_update、obligations、constraints、relationships
- 把 JSON 示例从"展示所有字段"改为"展示最小必填集"，附注"其余字段有变化时才加"
- 删除冗余的解释文字，只保留字段格式定义

### reviewer.md（263行 → 目标 80行）

问题：11个评分维度，模型平均分配注意力，每个都蜻蜓点水。

方案：
- 评分维度从11个砍到5个核心维度：
  1. 功能密度（每句话是否做了多件事）
  2. 角色辨识度（遮住名字能否认出是谁）
  3. 场景沉浸感（读者是否觉得自己在那里）
  4. 信息密度（有没有可以删掉的段落）
  5. 读者体验（读完想不想看下一章）
- 删除所有跟 style_gate 重复的检测（句长、段长、短句比例等——这些代码已经在查了，reviewer 不需要重复）
- 读者体验模拟的4个问题保留，但从"必须回答"改为"用1-2句话概括"

## 优先级

**策略调整：不做大重构，先做最小改动验证。**

大重构风险太高——砍掉规则后模型可能退化到默认AI腔，反而倒退。正确做法是一次改一个变量，验证有效再继续。

### 最小改动实验（立即执行）

只做一件事：在 writer.md 最顶部加一条"元原则"，不删除任何现有规则。

加的内容：
```
## 元原则（写每一句话之前过一遍）
每句话至少做两件事。一句话只推进剧情=流水账。好的一句话同时在做：推进事件+揭示性格，或建立氛围+埋线索，或传递情绪+提供新信息。写完一句，问自己：这句话在做几件事？只做了一件就改。
```

为什么只加这一条：
- 它是"功能密度"的直接指令，解决最大的质量差距
- 它不跟任何现有规则冲突
- 它是可验证的——对比加之前和加之后的章节，数每句话的功能数量
- 如果 mimo 能执行这条，说明精简方向是对的；如果执行不了，说明需要更具体的规则而不是更少的规则

### 验证方法

1. 加上这条后跑5章
2. 对比加之前的第1-2章和加之后的章节
3. 检查指标：
   - 是否出现"一句话只有动作没有别的"的比例降低？
   - 是否出现更多"一句话同时做两件事"的好句子？
   - 是否出现副作用（句子变长变复杂、过度塞信息）？
4. 如果正向 → 继续精简计划
5. 如果无效 → 问题不在规则数量，需要换思路（可能需要更好的示例、或换模型）
6. 如果负面 → 回滚，保持现状

### 后续路线（取决于实验结果）

- 实验成功 → 执行完整重构（精简 writer/archivist/reviewer）
- 实验失败 → 放弃精简方向，改为"加好例子"策略（在 chunk 里放2-3段精品示范文字，让模型模仿而不是遵守规则）
