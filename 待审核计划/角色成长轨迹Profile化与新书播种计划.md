# 角色成长轨迹 Profile 化与新书播种计划

> 状态：待审核。本文件只记录治本方案，未修改代码或运行产物。

## 结论

`config/growth_arcs.md` 是当前书主角成长与笔锋演变设定，但规划层把它当固定高优先级输入注入卷纲和弧线规划。换书若沿用默认配置，新书会继承当前书主角的心理、系统焦虑和成长节奏。

## 当前证据

- `scripts/pipeline/planning.py:620-622`：VolumePlanner 固定读取并注入 `config/growth_arcs.md`。
- `scripts/pipeline/planning.py:945-976`：ArcPlanner 同样固定读取并注入该文件。
- `config/growth_arcs.md:17`：写死“沈安的冷静是装的”。
- `config/growth_arcs.md:31`：围绕当前书系统心态展开成长阶段。
- `config/growth_arcs.md:82`：引用“北砚县和青石镇”的当前书经历。

## 根因判断

- “角色成长轨迹”是通用输入槽，但其内容是 book/profile 数据。
- 文件位于 `config/` 且被固定路径读取，容易被误认为通用配置。
- 新书初始化缺少自动生成/播种成长轨迹的流程，只能沿用旧文件。

## 影响

- 新书卷纲和弧线会被当前书心理轨迹污染。
- 规划师可能把不属于新书的主角裂缝、系统关系、地点经历当作硬设定。
- 若删除该文件，又会丢失“笔锋随角色成长”的通用能力。

## 治本方案

1. 拆分机制和内容。
   - 通用机制：成长轨迹输入槽、阶段 schema、笔锋演变维度。
   - 项目内容：具体主角、阶段、行为变化、禁忌，放入 book profile。
2. 新书播种流程。
   - 根据故事核、主角设定、目标篇幅自动生成初版 growth arcs。
   - 生成后进入待审核，不直接投入主流程。
3. 规划层读取 profile。
   - Volume/Arc 输入只读当前 book profile 的 growth arcs。
   - 未配置时注入通用 schema 需求，而不是旧书内容。
4. 增加污染扫描。
   - 换书 profile 下输入不得出现当前书角色/地点/机制词。
5. 增加测试。
   - 新书 profile 主角为 X 时，卷纲/弧线输入只出现 X 的成长轨迹。
   - 删除当前书 growth file 后，通用流程仍能提示需要生成而不是崩溃。

## 验收标准

- 成长轨迹能力保留，但具体内容随书切换。
- 新书初始化能生成待审核 growth arcs 草案。
- 规划输入不会把当前书成长文本当通用配置注入。
