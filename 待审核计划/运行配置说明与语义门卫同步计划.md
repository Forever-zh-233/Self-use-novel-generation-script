# 运行配置说明与语义门卫同步计划

## 结论

运行配置、README、测试维护说明和 `checks_test.py` 的实际检查范围已经发生漂移。默认章节批量、自动链路角色、上下文产物路径、JSON 配置覆盖范围等说明仍停在旧版本，容易让用户和测试都误判当前系统行为。

## 当前证据

- `config/run.json`：当前 `chapter_count=50`，`max_chapters_per_run=1000`。
- `config/run.json`：当前 `sleep_seconds_between_calls=1`，`per_chapter_retries=3`，`retry_backoff_seconds=45`，`request_timeout_seconds=240`。
- `scripts/run_pipeline.py:1702-1705`：实际按 `min(chapter_count, max_per_run)` 跑，注释也说明不再硬卡 5 章。
- `config/配置说明.md:82`、`README-使用说明.md:73`：仍说默认一次只生成 1 章。
- `config/配置说明.md:98`：仍说脚本内部最多 5 章。
- `config/配置说明.md` 没有解释单章为什么慢：固定 sleep、角色串行、FactChecker 修复链、角色内部截断/拒绝重试、整章重试退避分别是什么。
- `config/models.json` 默认大多数自动链路角色使用 `mimo-v2.5-pro`，provider 配置 `stream=false`，因此请求会等待完整响应返回。
- `config/配置说明.md:27-45`、`README-使用说明.md:90-93`：模型配置说明主要列 writer/reviewer/editor/archivist，漏掉自动主链会调用的 story_director、volume_planner、arc_planner、fact_checker、summarizer 等角色。
- `README-使用说明.md:195`：仍写上下文压缩产物在 `输出/context/`。
- `scripts/pipeline/core.py:101-111`：实际 role 目录映射为中文 `上下文`。
- `TESTING.md` 和 `测试文件维护指南.md` 描述 `check` 会守 JSON 配置。
- `tests/checks_test.py:123-160`：实际只解析 `models.json/run.json/chunks/index.json`，且 `run.json` 只断言少数浅字段。

## 根因判断

- 主流程从短跑/手动模式演进到长跑/自动规划链路，但说明文档和配置语义门卫没有同步升级。
- `check` 的职责说明写得比实际实现更宽，导致维护者以为配置目录已有全量守护。

## 影响

- 用户可能以为默认只跑 1 章，实际可能长跑 50 章。
- 用户可能把生成慢误判为“故意 sleep 很久”，但实际 sleep 默认只有 1 秒，主要耗时通常是多角色串行 LLM 调用和重试链。
- 配模型时可能只配四个角色，运行到自动规划/摘要/核查阶段才失败。
- 排查上下文压缩产物时按旧路径找不到文件。
- 新增或损坏 `structure_norms.json`、`strand_weave.json`、`travel_matrix.json` 等配置时，`check` 不一定提前发现。

## 治本方案

1. 更新运行说明。
   - 明确当前默认 `chapter_count/max_chapters_per_run`。
   - 把“最多 5 章”改成“由配置控制，有锁防并发”。
   - 增加“速度来源矩阵”：固定 sleep、角色串行、beat 额外检查、summarizer、fact_checker 修复链、archivist、压缩器、整章重试、角色内重试、analyst 并发/串行边界。
2. 更新模型角色说明。
   - 列出默认自动链路可能调用的全部角色。
   - 区分必配角色、可选/可跳过角色、一次性工具角色。
3. 更新产物路径说明。
   - 把 `输出/context/` 改为当前 `输出/上下文/` 或统一用 role_artifact 映射表描述。
4. 收紧 `check` 文档口径或增强检查。
   - 若保持“守 JSON 配置”说法，就解析 `config/*.json`。
   - 对核心配置增加最小 schema/语义断言。
   - 非核心配置至少保证 JSON 可解析、顶层类型符合预期。
5. 增加文档/配置一致性测试。
   - `run.json` 默认章节说明与实际配置不应长期冲突。
   - README 提到的核心目录应存在或由映射表生成。

## 验收标准

- 用户按 README/config 说明操作时，不会误判默认长跑规模。
- 默认自动链路角色在模型配置说明中完整出现。
- `check` 的承诺与实际检查范围一致。
