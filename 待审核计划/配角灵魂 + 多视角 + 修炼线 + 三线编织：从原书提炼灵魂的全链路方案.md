## 0. 这份计划要解决的真问题（一句话）

126 章读下来：**几乎全是主角视角，配角是工具人，没有信息差的张力，修炼线名存实亡（realm 卡在"凡人"到第130章）。** 而这不是 writer 写得差，是**规划层没要求、analyst 没提炼原书在这些维度上的灵魂、消费端的管子接了一半**。

**最高纪律（贯穿全篇，来自用户反复纠正的教训）：**
1. **绝不加计数式 KPI**（"每 N 章必须有配角高光""修炼篇幅必须占 X%"）——这会诱发 writer/beat 为过检硬塞戏，把工具人换成"为显得有灵魂而硬加戏的工具人"，是换马甲的同一个病。参见 [[anti-gaming-style-fix]]。
2. **先接管子，后灌料**：先确认消费端（prompt 注入位）存在并打通，再让 analyst 提炼原书。否则提炼出一堆手法没人吃。参见 [[verify-consumer-before-injecting]]。
3. **学原书的灵魂，不是通用手法**：analyst 每个新维度必须解剖"**这部原书具体怎么做到的**"（抽象成可复用动作，但提炼自真实文本），不是"配角要立体"这种正确的废话。
4. **配角灵魂的因果**：灵魂 = 这个配角**知道一些主角/读者不知道的事 + 基于此采取了自己的行动（有自己想瞒着办成的事）**。"高光时刻"是果，"隐藏议程 + 知识边界"是因。喂因，不考核果。

## 2. 总体架构：三层 × 五维度

```
原书(271702行) ──MAP扫读──→ 提炼"原书怎么写这5维" ──REDUCE──→ 手法chunk
                                                                  │
                                          ┌───────────────────────┘
                                          ▼ (注入消费端)
   arc_planner ──规划配角议程/修炼节奏/三线编织──→ beat_planner ──落到每章──→ writer
        │                                                                      │
        └── 写进 ledger 配角 arc_core/secrets ←── archivist ←──────────────────┘
```

**五个新维度**（analyst 要从原书学的灵魂）：
1. **配角塑造**：原书怎么让配角有独立生命（不是主角的摄像头）
2. **多视角/POV 切换**：原书什么时候切视角、切给谁、为什么这里非切不可
3. **信息差经营**（已 MAP 在抽、REDUCE 丢弃 → 救回来）：谁知道/谁不知道的落差怎么变成张力
4. **修炼线编织**：玄幻小说怎么让修为"有存在感"而不靠堆篇幅；瓶颈/突破/代价怎么写
5. **三线交织**：道途/情义/天地怎么编在一起呼吸，而非平均分配

## 3. 分阶段实施（按因果优先级，可独立交付）

### 阶段 A：修炼线急救（最高优先，含一个明确 bug）

**A1. 拆开道途线混合袋**
- `config/strand_weave.json`：把"道途线"拆成可测量的子维度。方案：保留道途线作大类，但新增**修炼子标记** `cultivation_active`（archivist 每章额外打一个布尔/强度标签：本章是否有实质修炼/境界/真气内容）。
- 这样 `strand_digest_for_director` 能同时报"道途线占比 60%（其中修炼实质含量：仅 8%）"——暴露空洞。
- **不设硬阈值**，只暴露数字给 story_director 定性判断（延续 soft_only 纪律）。

**A2. 境界进度追踪 + 停滞观察（bug 修复 + 新信号）**
- 先查清 realm 为什么卡在"凡人"：是 archivist 从不更新 realm 字段，还是突破事件没被记录。这是 bug，不是品味问题。
- 新增 `realm_progress` 追踪：当前境界、进入该境界的章号、该境界下的关键进展（如"叩门境真气走半寸"）。
- story_director 增加**定性观察**（非计数）：realm 多少章未动 + 该境界的瓶颈是否在被"啃"。像现有弧线审计那样写一段话，不做"必须 N 章突破"的机械判定。

**A3. prompt 接线**
- `arc_planner.md`：新增修炼线规划段——这条弧里修为往哪走（卡瓶颈/突破/学新法/受挫），境界进展的"啃"法。明确：修炼的存在感来自**进度可感知**，不是篇幅。
- `beat_planner.md`：① 把"推进的线"字段词汇与道途/情义/天地接轨（修复 vocabulary mismatch）；② 当 arc 标注本章有修炼进展时，beat 要给修炼一个**具体的身体锚点**（不是夹缝里提一句"他感到真气"，而是一个可写成场景的微事件）。

### 阶段 B：配角灵魂的入口（中优先，因）

**B1. arc_planner 规划配角议程**
- `arc_planner.md` 新增 `side_characters` 字段（每弧规划 **3-5 个主要配角**，不是每个路人）：
  ```json
  "side_characters": [{
    "name": "周济",
    "independent_goal": "他自己为什么留在青石镇——不依附主角的追求",
    "hidden_agenda": "他知道那包药有问题，但选择不告诉沈安（他有自己的盘算）",
    "knowledge_boundary": "知道X，不知道沈安失忆",
    "beat_moments": [{"ch": 93, "what": "有人来铺子找他，他让沈安回避——读者不知来者是谁"}]
  }]
  ```
- 关键：`hidden_agenda` + `knowledge_boundary` 是**信息差的源头**，这两个一旦存在，POV 的"暗流涌动""人物铺垫"就自然有素材。

**B2. archivist 把配角议程写进 ledger**
- `archivist.md`：当配角有 arc 规划的独立议程时，为该配角填充 `arc_core`（want/need/lie）和 `secrets`（含 known_by）——这些字段消费端已存在（B 节 1.1），只是空着。
- **不强制每个配角都填**，只填 arc 规划了的主要配角。

**B3. beat_planner 落到每章 + writer 注入**
- `beat_planner.md` 新增可选字段 `配角本章动作`：当主要配角出场时，写"该配角基于自己的议程在本章做了/瞒了什么"（不是"沈安觉得他怎样"）。仅 arc 标注了 beat_moments 的章必填，其余可选——**避免凑数**。
- `context.py`：普通主角章的 writer 上下文，注入在场主要配角的 `hidden_agenda`/`knowledge_boundary`（当前只有 POV 章注入 ignorant_of，主角章没有配角议程入口）。这是要新建的注入位。

**B4. POV 解禁 + 供给修复（双管）**

用户拍板（§7 问题3）：去掉"一弧最多 1-2 次"的硬上限，**完全交给弧线规划师判断**。但"解禁"和"会用"是两回事，要同时做两件事：

1. **去掉硬上限，换成判断力**（提示词工程）：
   - 改 `arc_planner.md:166`：删掉"一弧最多 1-2 次"的机械配额，改为"按叙事需要判断——该切就切，不滥用"。
   - 判断标准**来自 C 阶段从原书学到的多视角手法卡**（C 阶段 `chunk_多视角切换`）：什么节点该切、切给谁、切回来怎么发酵落差。没有这张卡，解禁等于让模型乱切——所以 B4 的"会用"依赖 C。
   - 双向防滥用：既防滥用他人视角（每章都跳走），也防滥用主角视角（什么都从沈安眼睛看）。这是定性引导，不设新的反向配额。
2. **修供给**（否则解禁也无米下锅）：
   - `impact_seeds` 一直空 → 影响回响型 POV 断粮。查链路：archivist 该产（archivist.md:301-306 有字段）、谁消费、断在哪（[[verify-consumer-before-injecting]]）。大概率是"实质影响"门槛太高、从没触发。
   - 暗流涌动/人物铺垫型 POV 靠配角的 `hidden_agenda`——由 B1 供给。B1 做完这两类自然有素材。

**依赖关系**：B4 的"解禁"动作本身轻量（删一句上限），但"解禁后能用好"强依赖 C 阶段的多视角手法卡。所以 B4 拆两步：删上限随 B 做；判断力提示词等 C 的 analyst 料到位后一起调。

### 阶段 C：analyst 五维度提炼（料）—— 改为尽早并行启动

> **执行顺序修正（据 §7 批注2/4）**：原计划把 C 放最后串行。但用户要求 A 的占比校准、B4 的 POV 判断、三线节奏**都借鉴原书**，且 analyst 重跑趁 API 免费要尽早。故 C 的 **MAP 重跑 + REDUCE 提炼**与 A、B 的代码改动**并行启动**（互不阻塞：A/B 改管线与 prompt 消费端，C 在产手法卡）。料产出后，再回头把 A3/B4 的提示词用原书手法"调准"。即：**接管子(A/B) ∥ 灌料(C)，最后用料校准 prompt**。

**C1. analyst_map.md 补三个维度的提炼指令**（信息差已有，补配角/POV/修炼/三线）：
- **配角塑造**：原书怎么让配角"离开主角也能活"——他自己的议程怎么通过行动（非主角转述）展示；配角的秘密/盘算怎么藏在对话潜台词里；路人怎么用一个"他的目光落在自己的事上而非主角"的细节带出即时利益。
- **多视角/POV 切换**：原书什么节点切视角、切给谁、切多久；切视角后怎么制造"读者知道而主角不知道"的张力；切回主角视角时怎么让这个落差发酵。
- **修炼线编织**：原书怎么让修为"有存在感"——是靠一句身体感知的锚点反复出现，还是靠瓶颈的"啃"，还是靠突破的代价；修炼怎么和剧情/情绪缝在一起而非独立成段；变强怎么换来新失去/新限制（已有"升级与代价"维度，强化到修炼具体层面）。
- **三线交织**：原书怎么让道途/情义/天地三条线在同一章里**互相缠绕**而非平均分配；一条线的事件怎么同时推进另一条线。

**C2. analyst_reduce.md 扩充产出 chunk**（当前只 3 张 → 扩充）：

- **注意 token 预算**：reduce 当前限"三张合计 ≤2000 token"，扩充后要分批 reduce 或放宽（写手不会同时读全部，靠 select_chunks 关键词点亮）。

**C3. context.py select_chunks 注册新卡关键词**：
- `chunk_配角塑造` ← 关键词：配角名出现、"出场角色"含非主角主要角色
