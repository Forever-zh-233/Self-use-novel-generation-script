# 重复动作签名 Pattern Profile 化计划

> 状态：待审核。本文件只记录治本方案，未修改代码或运行产物。

## 结论

重复动作检测是通用 Writer 质量机制，但默认 `SIGNATURE_PATTERNS` 混入当前书物件和角色动作。换书后，该机制可能注入当前书标签，也无法检测新书自己的标志性重复动作。

## 当前证据

- `scripts/pipeline/context.py:25-34`：`SIGNATURE_PATTERNS` 写死“竹杖”“黑子”“耳朵压平”“鼻子拱”等当前书相关动作。
- `scripts/pipeline/context.py:1168`：扫描近章正文并匹配这些签名。
- `scripts/pipeline/context.py:1258`：匹配结果作为“近期重复动作禁用清单”注入 Writer。

## 根因判断

- 反重复机制本身通用，但 pattern 表属于 book/style profile。
- 当前代码把通用抽象模式和当前书具体载体混在同一个常量里。
- 新书没有生成自己的 signature pattern 初始化流程。

## 影响

- 换书后 Writer 输入可能出现当前书物件/角色标签。
- 新书的高频动作、口癖、意象重复无法被检测。
- 反重复机制可能既污染又失效。

## 治本方案

1. Pattern 分层。
   - 通用 pattern：停顿、沉默、手抖等跨书抽象表达。
   - Book pattern：角色、物件、物种、专属动作，来自 profile。
2. 自动发现与待审核。
   - 从已生成正文/Analyst 结果中发现高频动作签名。
   - 新 pattern 先进入待审核，不直接生效。
3. 注入文案去实例化。
   - Writer 看到的是“近期重复动作禁用”，不应引入不属于本书的对象。
4. 增加换书测试。
   - 无当前书 profile 时，输入不出现“竹杖/黑子”等当前书标签。
   - 新书 profile pattern 能正常触发提醒。

## 验收标准

- 重复动作检测表可随书切换。
- 通用代码不内置当前书专属物件和角色动作。
- 新书能生成并审核自己的重复签名表。
