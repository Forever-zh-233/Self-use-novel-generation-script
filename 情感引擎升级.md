# 情感引擎升级（回响 + 冰山 + 情感技法卡）

基于网络调研的可编码情感技法，落地三个系统。核心约束：不爆 token、不抄原文、有界增长。

## 系统一：情感锚点账本（回响/草蛇灰线）

**数据结构**（ledger.json 新增 `emotional_anchors`）：
```json
{
  "id": "EA-001",
  "type": "种子/仪式/承诺/失去/意难平",
  "chapter": 3,
  "content": "沈安给张寡妇孩子退烧,孩子说'婆婆说别走'",
  "object": "孩子的笑/那块湿布",   // 可回响的具体物件,可空
  "emotional_target": "牵挂/愧疚",
  "echo_status": "活跃/已回响/沉睡",
  "last_echo_chapter": null,
  "echo_count": 0,
  "note": "可在主角多年后路过此地时回响"
}
```

**增长控制**：
- 活跃锚点上限 30（超了把最久没回响的转沉睡）
- 已回响 2 次以上的降权
- 意难平类永久保留（这是全书最高价值的情感债）

**archivist 职责**：每章归档时,判断本章有没有产生"情感分量时刻"（告别/承诺/失去/牵挂/未竟之事）,有则登记 EA。大部分章节没有,留空。

**beat_planner 职责**：规划时能看到"可回响的活跃锚点列表",在合适的时机（时间跳跃、故地重游、相似情境）安排回响。回响要满足研究里的 EARN 原则——不是硬塞,是自然触发。

**writer 注入**：当 beat 标注了"回响 EA-XXX"时,注入该锚点的内容 + 冰山指令"不要直接提这件旧事,用一个物件/动作/半句话让它自己浮上来"。

## 系统二：冰山分层（writer 看到的信息分水上/水下）

**问题**：现在 ledger 把 secrets/未公开真相全给 writer,writer 倾向于全写出来。

**改造 `ledger_context_for_writer`**：把信息明确分两层标注——
- 【水面上·本章可外显】：本章 beat 允许写的事实
- 【水面下·你知道但本章绝不能说破】：secrets、未到外显时机的伏笔、读者还不该知道的真相。明确告诉 writer:"这些是冰山水下,你知道全貌,但本章只能让它影响角色的反应和选择,一个字都不能写出来。"

这样 mimo 有了真正的冰山——知道全貌但被强制克制。这是我们多 agent 系统独有的优势。

**实现**：secrets 字段、长线伏笔未外显部分,单独归到"水面下"section,加强约束文字。

## 系统三：情感技法卡（3张新 chunk）

不重跑分析师,直接做成方法论卡片（普世 craft,非原文归纳）。

### chunk_情感回响手法.md
- 草蛇灰线：早期低调埋种子,后期原样重现,靠"物没变人变了"的落差
- 同景不同情：同一场景/动作演两次,情绪反转,前一次是后一次的尺子
- 移情于物：一个物件反复出现累积意义,最后一次"减去那个人"
- 空椅子：写缺席的形状(没人用的碗、半边的床),absence 比 presence 更重
- 延迟的悲伤：不在失去当时崩,几章后被一个小触发点击中

### chunk_克制与留白.md（冰山执行）
- 峰值禁止命名情绪（不写"悲痛欲绝",写身体的微动作）
- 省略中心：写痛苦事件的周围(聊天气、聊吃饭),那件事一字不提
- 移情于身体：grief 写成肢体动作(反复掸裤子上的灰、盯着墙上一块污渍)
- 早切一拍：情绪场景在读者期待的解决前一拍结束,留白给读者
- 低反应：事越大,角色当场反应越小(反比例),读者补上情绪
- 白描：用平、冷、不加修饰的句子写最痛的事,平淡对内容的反差就是效果

### chunk_意难平.md（网文最高价值情感）
- 全书设定1-2条"意难平"线:本该圆满却差一点的关系/没说出口的话/几乎避开的命运
- 保护它不被轻易收束,withhold resolution
- 不是悲剧,是"差一点"——读者放不下的那口气

## 实施顺序

1. 先做 3 张技法卡（纯文件,零风险,立即可用）
2. 改 `ledger_context_for_writer` 做冰山分层（中等风险,需测试）
3. 加 emotional_anchors 数据结构 + archivist 登记 + merge 逻辑（较大,仔细测）
4. beat_planner 加"回响时机"字段 + writer 注入回响指令
5. select_chunks 接入新技法卡（按需:有回响/情感峰值时注入）

## Token 预算
- 3张卡：按需注入,每张 ~400 tokens,一次最多注入1-2张
- emotional_anchors：context 只注入"活跃且本章相关"的,~150 tokens,有上限
- 冰山分层：不增加 token,只是把已有信息重新分组标注
- 净增：~300-600 tokens/章（按需）,远低于上限

## 验证
- 每步 py_compile
- emotional_anchors 迁移后验证 JSON
- 跑1章确认新字段被 archivist 填、被 writer 用、不报错
