# 精品逼近升级计划（四路网络调研 → 缺口落地）

目标：在不换 mimo 模型、短期不爆 token、长期不膨胀的前提下，补齐"逼近精品文"缺失的机制。
原则：复用现有架构（手法卡 + 按需注入 + 账本 + gate + story_director），新机制一律走"存在才点亮/相关才注入"，常驻成本≈0。

调研收敛出的缺口分三梯队落地。每个 Phase 独立可验证（dry-run + 跑 1 章）。

## 本轮决策（用户拍板）
- **手法卡先手写、以后用 analyst 覆盖**：Phase 1 的卡这轮我手写（立刻能用，零 token）。同时给 `prompts/analyst_map.md` 加新提炼维度，作为"未来重跑覆盖"的预埋——下次全量分析会自动产出从原著（287万字同类瞎子修仙文）学来的高质量卡，覆盖手写的"通用常识"版。手写=临时占位，analyst=未来正式版。
- **账本与系统层机制全做**：Phase 2/3 这轮一并落地。
- **Best-of-N（3.3）默认只在关键章/高潮章开启**（费 token，平推章不开）。

## Phase 1 — 手法卡 + gate（Tier 1，零结构改，几乎不增 token）

最高 ROI、最低风险。全是加卡 + 关键词点亮 + gate 软警告，不碰任何账本结构。

### 1.1 盲眼感官红利（最大护城河，三路收敛）
- 新增 `prompts/writer_modules/盲感官.md`，**默认注入**（像现有"视觉"模块那样每章给）。
- 内容合并三层（都源于"主角是盲人"）：
  - 非视觉深 POV：每场景至少 2 个非视觉感官锚点（声/嗅/触/温度/空间回响/把脉/药味）；禁用"看见/望去"作主感知。
  - 陌生化（Shklovsky）：关键物件/事件至少一次绕开惯常感官与惯常比喻重新呈现；盲感官天然制造"像第一次看见"。
  - 意境/情景交融（王国维有我/无我之境）：让景的冷/空/静直接是没说的心境，每个"愿"收束尽量落在纯景物空镜。
- 在 `writer_focus_modules()` 里把"盲感官"加进默认注入（与"视觉"并列）。
- gate（`style_gate`）加**软警告**：统计非视觉感官词 vs 视觉动词比例，过度视觉主导只 warn 不卡。

### 1.2 主角能动性 + Try-Fail + 有代价的胜利
- 新增 `chunks/chunk_主角能动性.md`：每场戏先定"沈安此刻主动想要什么+做了什么决定"；结局只许"是,但…/否,而且…"，禁止"是,然后顺利"；关键打脸前必有一次铺垫性挫败；胜利必须有代价。
- `select_chunks` 关键词点亮（冲突/选择/决定/失败/代价/挣扎等）。
- `prompts/beat_planner.md` 加引导字段："本章主角主动想要"，提示规划师标注（不强制每章）。

### 1.3 具体性原则 + 设定戏剧化投放
- 新增 `chunks/chunk_具体与投放.md`：抽象阶梯（"很多/一些/某种/气氛"→下沉到具名实物/精确数量）；设定禁止连续大段讲解，必须藏进当下冲突/需求里渗透给出。
- `select_chunks` 关键词点亮（设定/解释/来历/世界观相关时）。

### 1.4 信息差 / 戏剧性反讽
- 新增 `chunks/chunk_反讽落差.md`：区分戏剧性反讽（读者知道、角色不知道→揪心）与处境反讽（结果与意图相悖→苦涩）；扮猪吃虎/打脸的底层开关；反转场景标注三方知情度落差。
- `select_chunks` 关键词点亮（反转/隐瞒/误会/真相/装/不知道等）。

### 1.5 场景-后续 + 微张力
- 新增 `chunks/chunk_后续与微张力.md`：重大冲突后强制插入"后续"（反应→两难→新决定）这一呼吸单元，防 AI 跳过消化导致情感扁平；关键段落让角色始终带一组矛盾情绪，填满章末钩子之间的平地。
- `select_chunks` 关键词点亮（冲突/打斗/噩耗/重大事件后续相关）。

### 1.6 两条免费加固（不新增卡）
- `chunks/chunk_克制与留白.md` 卡头加入 Eliot"客观对应物"统摄原理（解释为何禁止命名情绪——情绪只能靠对应物的公式逼出来）。
- `chunks/chunk_意难平.md` 加"意难平三要素 = 依恋 + 利害 + 意料外的必然"前置检查（没有前置依恋，克制写法=读者没感觉）。

### 1.7 收尾
- 所有新卡登记 `chunks/index.json`（含 tokens 估算、category）。
- 新卡写完跑 `scan_chunk_for_contamination` 确认零专名零原句（沿用现有铁律）。

### 1.8 预埋 analyst 新维度（为未来覆盖做准备，本轮只改 prompt 不重跑）
- 给 `prompts/analyst_map.md` 的"重点盯的维度"表新增几条，让下次全量分析能自动产出对应手法卡：
  - **非视觉感官承载**：作者如何用听/触/嗅/温度/空间回响代替视觉来写画面（原著主角也是瞎子，范例满篇）。
  - **Try-Fail / 有代价的胜利节奏**：胜利前怎么先败、打脸前怎么憋挫败、变强怎么付代价。
  - **信息差 / 戏剧性反讽**：扮猪吃虎、读者知道角色不知道的落差怎么经营。
  - **场景-后续呼吸单元**：重大冲突后怎么写消化/两难/决定，而非一个冲突接一个。
- 这步只改提炼框架，不触发重跑。下次跑 `--analyst` 时自动产出，接进 select_chunks 覆盖手写卡。
- ⚠️ 重跑成本：上次全量分析 61 批 ×40k 预算量级。重跑时机由用户在有 token 时另行决定。
- **手写卡与 analyst 卡的关系**：手写=临时占位（通用写作常识），analyst=未来正式版（287万字真实手感）。覆盖时同名卡 analyst 版优先。

**Phase 1 token 影响**：默认注入只多 1 张盲感官模块（≈250 token）；其余卡按需点亮，不相关章节不进。常驻几乎 0 增长。

## Phase 2 — 账本机制 + story_director 维度（Tier 2，小 token）

涉及 ledger 结构、merge 逻辑、archivist delta、注入、director 维度。改动顺着现有 merge_ledger_update 范式。

### 2.1 主题论辩账本 + 角色自欺
- `ledger.json` 新增 `thematic_stances[]`：每个核心主题问题 + 各配角代言的不同答案 + dignity + tested_in + verdict（多数 NEVER_RESOLVE）。
- entity 加 `self_deception` 字段：{lie, contradicted_by[], status}。首推沈安（治心病的郎中患最重的心病）。
- `merge_ledger_update()` 加 handler：thematic_stances_update / self_deception_update。
- `prompts/archivist.md` 加对应 delta 字段 + 示例。
- 新增 `chunks/chunk_主题对位.md`（反方必须由读者尊重的角色说出，本卷内不裁决）+ `chunks/chunk_自欺与道德灰度.md`（自欺绝不说破，只靠行动反驳；沈安每卷至少一个让读者皱眉的选择）。
- writer 按需注入（beat 标注主题折射/困境时，带上相关 stance + 沈安 self_deception 作水下信息）。
- story_director 加维度："本 5 章是在论证主题还是只在堆事件 / 沈安立场是否被某角色真正撼动过"。

### 2.2 战力 / 升级 / 代价治理（修仙头号死因）
- 复用已有 `threat_ladder.json` + `power_scaling.json`（已接进 beat）。
- 新增 `chunks/chunk_升级代价.md`：broadening（铺开场面）与 deepening（拧紧角色螺丝）交替，禁止连续纯数值升级；每次变强换来新代价/新失去/新两难；盲眼本身是天然代价设定。
- `select_chunks` 关键词点亮（突破/境界/变强/升级相关）。
- story_director 加维度："本卷强度提升是否只是更大数字？角色处境是否真的更难了？"

### 2.3 母题意义生长（修已有账本的空 meaning）
- `motifs[]` 加 `kind`（线索/主题意象）字段；主题意象强制每次复用写下"这次新增了什么意义"（evolution 已有，补 meaning 非空校验）。
- `merge_ledger_update` motifs handler 支持 kind + meaning 更新。
- `prompts/archivist.md`：motifs_update 指令要求主题意象每次复用必须给 meaning 增量。
- 在 `09-故事核.md` 锁定 2-3 个主题意象成系统（建议 盲/见、灯火/灰烬）。

### 2.4 角色弧内核 Want/Need/Lie/Truth
- entity 加 `arc_core`：{want, need, lie, truth, 转折点[]}（主角必填，主要配角选填）。
- `character_arcs_for_writer` 注入时带上本章出场角色的 arc_core。
- story_director 维度补："本卷主角是否有行为级变化，还是只有嘴上成长"。

**Phase 2 token 影响**：thematic_stances/arc_core 只在相关章注入；motif 已在注入。新增常驻≈0，相关章 +几百 token。

## Phase 3 — 系统层（Tier 2/3，费 token 或结构改，分步做）

威力大但要么费 token、要么动 ledger 结构。每条独立，做完一条验证一条。

### 3.1 生成时检索注入 + ledger 时间有效性（根治中段穿帮）
- entity 加 `valid_from / valid_to / state_at`（带时间区间的结构化状态）。
- writer 动笔**前**：按本章 beat 实体清单，从 ledger 精准召回"截至上一章的正确状态"作为硬约束模块注入（SCORE 思路，顺着现有按需注入哲学）。
- `fact_checker` 从"读正文比对"升级为"查状态表比对"，专攻 temporal 穿帮。
- 需配套 `scripts/migrate_ledger.py` 给存量 entity 补字段。

### 3.2 线索/支线台账 + 揭示节奏台账
- `ledger.json` 新增 `threads[]`：每条线 {状态:活跃/休眠/已收, 上次推进章, 计划收束章, 负责角色}；`reveal_ledger[]`：每大设定 {作者已知, 已揭示程度, 计划在哪卷揭下一层}。
- story_director 维度："有无线索休眠过久 / 开出去的线有无收束计划 / 世界观揭示是否超配额"。
- 防 800 章断线、烂尾、神秘感提前破产。

### 3.3 Best-of-N 候选择优（榨干 mimo 的唯一办法，但费 token）
- writer 同 prompt 采样 N 份（N=3~4），用**已有的 gate + reviewer 分**当排序目标，候选里加"与已写章节情节相似度"惩罚（反自我重复，Sui Generis 思路）选 top-1。
- ⚠️ **token 成本**：writer 这步 ×N，是每章最贵环节。**默认只在关键章/高潮章开启**，平推章不开，由 run.json 配置 + beat 标注触发。
- 这是不换模型下逼近 mimo 能力上限的唯一手段，也是"创造性"的系统级答案。

#### 3.3+ 自动触发设计（2026-06-01 用户共识，补"怎么自动在高潮章开"）

**问题**：原 3.3 的 `best_of_n_enabled` 装了开关但没接感应器——触发靠 run.json 手敲 `key_chapters`（800章不现实）或 beat 的"高潮章"字段（schema 里根本没这字段，永远空）。所以现状是"开了也不会自动在高潮章触发"。

**诚实评估（先说清，别神化）**：Best-of-N 是"取上限"不是"抬上限"——扔4次骰子取最大点，扔不出7。它能把 mimo **偶然**写出的神来之笔捞出来（针尖类高光有概率收益），分数从53→55/56现实；但"主角裂缝""作者指纹"这类**不在 mimo 能力分布内**的东西，跑40次也出不来。**定位=高潮章的质量保险（托底+碰运气），不是"开了就爆"。** 真正的语感天花板仍只能靠换 writer 模型。

**触发机制（用户最终设计：beat_planner 决策 + arc 护栏 + director 建议）**：
- **arc_planner 划护栏**：arc 节点已带 `tension` 字段（高/高潮=爆点）。代码读节点 tension，把"高潮节点章 ±1"定为**允许开启的窗口**。这是结构意图，防止 beat_planner 在随便哪章乱标关键章骗资源（Goodhart 防护）。
- **beat_planner 做决策**：它每章跑、在 writer 前跑、最清楚这章是铺垫还是引爆。它看到"我在弧线爆点附近"+"director 建议临近高潮"，自己判断"这章是不是真要爆"，在 beat 里标 `关键章: true`。**决策权在最该有的人手里——它就是决定这一章干什么的人。**
- **story_director 只建议**：它巡查时可以在 beat_priority/arc_instruction 里提"临近高潮"，供 beat_planner 参考。**不做门禁、不做决策**——审计员不该当操盘手。
- **代码读 beat 的 `关键章` + arc 窗口护栏**：`beat.get("关键章") AND in_climax_window(chapter)` → 开 Best-of-N。窗口外标了不算。
- **run.json key_chapters 手动白名单保留**：后门，人工指定不受窗口限制（你知道自己在干什么时用）。

**为什么比"director 双签"更好（用户推翻旧方案的理由）**：
1. 无时序问题：beat_planner 在 writer 前跑，标完关键章 writer 还没动笔，Best-of-N 来得及。director 做决策有"高潮写完了还要他干啥"的洞。
2. 高潮漂移自动解决：beat_planner 每章实时判断，不存在"计划说36章爆但实际38章才爆"的问题。
3. director 回归本职：只建议不限制，与"故事总监批注是方向盘不是打卡表"一致。
4. 不需要强制 director 每章跑（省 token）：director 照旧 5 章一跑，它的建议是参考不是门禁。

**实施落点**：① `best_of_n_enabled` 改为读 beat `关键章` + `in_climax_window` 护栏；② `in_climax_window()` 新函数读 arc 节点 tension；③ beat_planner.md 加"关键章标记"指引（什么时候标/什么时候不标/参考 director 建议）；④ normalize_beat 保留 `关键章` 字段；⑤ run.json 仍留 enabled 总开关（默认 false）。**已实施并验证。**


**Phase 3 token 影响**：3.1/3.2 是按需召回 + 账本，小幅增长且有界；3.3 关键章 ×N，可控（仅高潮章）。

## 诚实的天花板
上述全部把 mimo 的下限托高、穿帮和套路压下去，但**句子能多漂亮，最终仍卡在 mimo 模型本身**（DeepMind Agents' Room：纯 prompt 编排提升有限，微调过的写作 agent 才显著超基线）。Best-of-N 是不换模型下的最优解，换 writer 到更强模型仍是回报最高的单笔投资——本次按用户决定暂不换。
